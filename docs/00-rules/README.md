# ローカルルール・開発規約

このディレクトリには、Databricks IoTシステムプロジェクトの開発における各種ルールや規約が格納されています。
これらのルールは、開発ツール（Cursor AI等）やチームメンバーが参照し、一貫性のある開発を行うためのものです。

## 📋 ドキュメント一覧

### cursor-rules.md (作成予定)
Cursor AIが参照するプロジェクト固有のルール
- AIアシスタントへの指示
- コード生成時の制約事項
- プロジェクト固有の命名規則

### coding-standards.md (作成予定)
Databricks IoTシステムのコーディング規約
- Python/Flaskコーディング規約
- Flask/Jinja2ベストプラクティス
- ファイル・フォルダ構成ルール

### project-conventions.md (作成予定)
プロジェクト全体の規約
- Git運用ルール（ブランチ戦略、コミットメッセージ）
- 環境変数の管理方法
- セキュリティポリシー

### e2e-testing-rule.md
Playwrightを利用したE2Eテストのルール
- 既存のE2Eテストスイートの概要
- 環境セットアップと実行手順
- テスト実装時のベストプラクティス

### flask-jinja2-rule.md
Flask + Jinja2 実装のルール
- View関数とテンプレートの責務分離
- サーバーサイドレンダリング（SSR）の活用
- フォーム実装（Flask-WTF）とバリデーション
- テンプレート再利用戦略（マクロ、インクルード、継承）

### css-design-token-rule.md
CSSデザイントークン管理のルール
- CSS変数（Custom Properties）によるトークン管理
- 直接値指定の禁止
- トークン追加・参照ルール

## 🔧 使用方法

### Cursor AIでの利用
1. `.cursorrules`ファイルを作成し、cursor-rules.mdの内容を参照
2. プロジェクトルートに配置することで、Cursor AIが自動的に認識

### 開発者向け
1. 新規参画時は全てのドキュメントを必読
2. コードレビュー時の基準として活用
3. 不明点がある場合は、これらのドキュメントを参照

## ✏️ ルールの更新

ルールの追加・変更が必要な場合：
1. 該当するドキュメントを更新
2. 変更履歴を記載
3. チームメンバーに周知
4. 必要に応じてレビューを実施

## 📌 重要な原則

- **一貫性**: プロジェクト全体で統一されたスタイルを維持
- **可読性**: コードは書く時間より読む時間の方が長い
- **保守性**: 将来の変更に対応しやすい設計
- **セキュリティ**: セキュアコーディングの実践
