# Flask + Jinja2 実装ルール

## サーバーサイドレンダリングの活用
- Jinja2テンプレートエンジンによるサーバーサイドレンダリングを基本とし、初期表示の高速化を図ります。
- データ取得や初期レンダリングはサーバー側で行い、不要なクライアントサイドJavaScriptを避けます。
- 動的な操作が必要な場合のみ、最小限のJavaScriptを使用します。

## 責務分離とコンポーネント構造
- **View関数**では**データ取得とビジネスロジック**のみに責務を絞り、HTMLレンダリングはJinja2テンプレートに委譲します。
- **テンプレート**は**表示のみ**に責務を絞り、ビジネスロジックを含めないようにします。
- 共通UIパーツは`templates/components/`や`macros/`に整備し、再利用を促進します。
- Blueprint単位で機能を分割し、役割が即座に分かる命名を行います（例: `admin/`, `alert/`, `dashboard/`）。

## ビジネスロジックとデータ取得
- ページ固有のロジックや複雑な処理は専用のサービス層やユーティリティ関数へ切り出してください。
- View関数から直接SQLを発行せず、Repository層やORM経由でデータアクセスを行います。
- 複数のView関数で共有する処理は`app/utils/`や`app/services/`配下に配置し、View関数から直接実装しないようにします。

## テンプレート構造
- ベーステンプレート（`base.html`）を継承し、ブロック（`{% block content %}`）で個別ページを実装します。
- 共通パーツは`{% include %}`で分離し、繰り返し利用する要素はマクロ（`{% macro %}`）で定義します。
- テンプレート階層は最小限に保ち、派生値はView関数側で計算してコンテキストに含めます。

## フォーム実装
- フォームは `Flask-WTF` と `WTForms` を組み合わせ、型安全かつバリデーションを一元化します。
- View関数内で個別にバリデーションするのは避け、フォームクラスでルールを定義します。
- バリデーションスキーマは`forms.py`など専用ファイルに分離して再利用性を高めます。
- 全てのフォームにCSRFトークン（`{{ form.csrf_token }}`）を含めます。

## エラーハンドリング
- エラー表示は`@app.errorhandler`でカスタムエラーページ（`errors/404.html`等）を実装します。
- ユーザー向けメッセージはフラッシュメッセージ（`flash()`）で通知し、テンプレート側で`get_flashed_messages()`で表示します。
- 例外はView関数側で分類し、UIにはユーザー向けメッセージのみを渡すようにします。

## テンプレート再利用戦略
- フォームフィールド、ページネーション、カードなど共通パターンを`macros/`に抽出し、重複を防ぎます。
- 共通マクロは明示的なパラメータを定義し、使用箇所で`{% from %} {% import %}`で読み込みます。
- テンプレート数が増えたらフォルダ階層（`templates/admin/`, `templates/alert/`など）を整理し、検索性を高めます。

## セキュリティ対策
- XSS対策: Jinja2の自動エスケープを活用し、`| safe`フィルタは特別な理由がない限り使用禁止です。
- CSRF対策: 全てのフォームに`{{ form.csrf_token }}`を含めます。
- SQLインジェクション対策: パラメータ化クエリを使用し、文字列連結でのSQL構築を禁止します。
- 静的ファイル参照: `url_for('static', filename='...')`を使用し、ハードコードされたパスを避けます。

## 実装フローの推奨ステップ
1. 仕様・要件を`docs/`で確認し、必要なデータとUI責務を洗い出す。
2. Blueprint骨組みを定義し、URLルーティングを設計する。
3. フォームクラス（`Flask-WTF`）を作成し、バリデーションルールを整理する。
4. View関数でデータ取得・ビジネスロジック・コンテキスト渡しを実装する。
5. Jinja2テンプレートでHTMLを作成し、ベーステンプレートを継承する。
6. 共通パーツやマクロで再利用可能なコンポーネントを整理する。
7. エラーハンドリングとフラッシュメッセージを実装する。

以上の原則に従い、Flask + Jinja2の恩恵を最大化しつつ保守性の高いコードベースを維持してください。
