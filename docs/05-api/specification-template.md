**注:**
- 入力先の「区分」には、パラメータの種類を記載します（リクエストボディ/パスパラメータ/クエリパラメータ）
- パスパラメータ: リソースを一意に識別するID（例: `/users/{userId}`）
- クエリパラメータ: 検索・絞り込み・ページング・ソート（例: `?name=山田&page=1`）
- リクエストボディ: POST/PUT/PATCHで送信するJSONデータ
- 物理名は基本的に**キャメルケース**で記述（例: `userId`, `groupName`, `createdAt`）
- ただし、**クエリパラメータ**はREST APIの慣習に従い**スネークケースも許容**（例: `per_page`, `sort_by`）
- **レスポンスJSON**のキーは必ず**キャメルケース**で統一（例: `perPage`, `sortBy`）
- 詳細は[APIガイド - REST API設計指針](./api-guide.md#2-rest-api設計指針に従う)を参照してください

### 4.1 レート制限超過時のレスポンス

レート制限を超過した場合、以下のレスポンスが返されます：

**HTTPステータス:** 429 Too Many Requests

```json
{
  "success": false,
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "リクエスト制限を超えました。しばらく待ってから再度お試しください"
  },
  "timestamp": "2025-11-01T10:00:00+09:00"
}
```

**レスポンスヘッダー:**

```http
X-RateLimit-Limit: {制限値}        # 制限値
X-RateLimit-Remaining: {残数}      # 残りリクエスト数
X-RateLimit-Reset: {リセット時刻}  # リセット時刻（Unixタイムスタンプ）
```

**記入例:（テンプレート参考用 - 実際の仕様書では削除）**

```http

X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1730390400
```

## 5. スコープ制限

このAPIのデータアクセススコープ制限を記載します。

### 5.1 概要

本システムでは、JWTトークンに含まれる `contract_id`（契約ID）に基づいて、ユーザーがアクセス可能なデータを制限しています。

**基本ルール:**
- ユーザーは自身の契約IDに紐づくデータのみアクセス可能
- 他の契約に紐づくデータにはアクセスできません
- 契約IDはJWTトークンから自動的に取得され、サーバー側で検証されます

### 5.2 このAPIのスコープ制限

| 項目 | 制限内容 |
|------|---------|
| アクセス可能なデータ | {このAPIで取得・操作できるデータの範囲} |
| 制限方法 | {どのように制限しているか（WHERE句、JOIN条件など）} |
| 例外条件 | {制限が適用されない場合（あれば）} |

**記入例:（テンプレート参考用 - 実際の仕様書では削除）**

| 項目 | 制限内容 |
|------|---------|
| アクセス可能なデータ | ログイン中のユーザーの契約ID（`cnt_001`）に紐づくユーザー情報のみ |
| 制限方法 | SQL WHERE句: `contract_id = :contractId`（JWTから取得） |
| 例外条件 | `super_admin` ロールは全契約のデータにアクセス可能 |

### 5.3 JWT構造例

```json
{
  "sub": "usr_001",
  "name": "山田太郎",
  "email": "yamada@example.com",
  "contract_id": "cnt_001",  // ← この契約IDに紐づくデータのみアクセス可能
  "role": "system_admin",
  "exp": 1730390400,
  "iat": 1730386800
}
```

### 5.4 スコープ制限エラー時のレスポンス

他の契約のデータにアクセスしようとした場合、以下のレスポンスが返されます：

**HTTPステータス:** 403 Forbidden

```json
{
  "success": false,
  "error": {
    "code": "PERMISSION_DENIED",
    "message": "この操作を実行する権限がありません",
    "details": {
      "reason": "契約IDが一致しません"
    }
  },
  "timestamp": "2025-11-01T10:00:00+09:00"
}
```

**注:** スコープ制限の詳細は[認証・認可](./common-specification.md#8-認証認可)を参照してください。
