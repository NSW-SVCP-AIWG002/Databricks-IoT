## 1. 基本情報

| 項目 | 内容 |
|------|------|
| エンドポイント | `GET /users` |
| 説明 | 契約に紐づくユーザー一覧を全件取得する。検索・フィルタリング・ソート・ページングは全てクライアント側で実施 |
| 認証 | 必要 |
| 権限 | `system_admin`, `operation_admin` |

**注:**
- エンドポイントは[ベースURL](../../common-specification.md#1-ベースurl)からの相対パスで記載します
- 権限には、このAPIを実行可能なロールを記載します
- ロールの詳細は[認証・認可](../../common-specification.md#8-認証認可)を参照してください
- データ形式（JSON、UTF-8）: [データ形式](../../common-specification.md#2-データ形式)参照
- 日時フォーマット（ISO 8601、JST）: [日時フォーマット](../../common-specification.md#13-日時フォーマット)参照
- スコープ制限: [スコープ制限](#5-スコープ制限)参照

## 2. 入力仕様

### 2.1 入力パラメータ詳細

| No | 論理名 | 物理名 | 属性 | 必須 | 文字列 |  | 数値 |  | 入力先 |  |  | 備考 |
|----|--------|--------|------|------|--------|--------|------|------|--------|-----------|------|------|
|  |  |  |  |  | 最小桁 | 最大桁 | 最小値 | 最大値 | 区分 | オブジェクト | 項目 |  |
| 1 | 契約ID | contractId | 文字列 | ○ | 36 | 36 | - | - | JWTトークン | - | contract_id | Authorizationヘッダーから自動抽出。UUID形式 |

**注:**
- このAPIはクエリパラメータを受け付けません
- 契約IDはJWTトークンのペイロード（contract_id）から自動的に取得されます
- サーバー側では契約IDによる絞り込みのみ実施し、全件データを返却します
- 検索・フィルタリング・ソート・ページングは全てクライアント側で実施してください
- **レスポンスJSON**のキーは**キャメルケース**で統一
- クライアント側キャッシュ戦略により、全件取得してクライアント側でフィルタリング・ページング・ソートを行う
- 詳細は[API共通仕様書 - クライアントサイドキャッシュ戦略](../../common-specification.md#9-クライアントサイドキャッシュ戦略)を参照

### 2.2 入力サンプル

**全件取得（契約IDで自動絞り込み）:**
```
GET /users
```

## 4. レート制限

このAPIには以下のレート制限が適用されます：

| 項目 | 制限値 |
|------|--------|
| 制限値 | 共通制限を適用（1,000リクエスト/時） |
| 適用範囲 | ユーザー単位 |

**注:**
- デフォルトのレート制限は [API共通仕様書 - レート制限](../../common-specification.md#12-レート制限) を参照してください

### 4.1 レート制限超過時のレスポンス

レート制限を超過した場合、以下のレスポンスが返されます：

**HTTPステータス:** 429 Too Many Requests

```json
{
  "success": false,
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "リクエスト制限を超えました。しばらく待ってから再度お試しください"
  },
  "timestamp": "2025-11-01T10:00:00+09:00"
}
```

**レスポンスヘッダー:**

```http
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1730390400
```

## 5. スコープ制限

このAPIのデータアクセススコープ制限を記載します。

### 5.1 概要

本システムでは、JWTトークンに含まれる `contract_id`（契約ID）に基づいて、ユーザーがアクセス可能なデータを制限しています。

**基本ルール:**
- ユーザーは自身の契約IDに紐づくデータのみアクセス可能
- 他の契約に紐づくデータにはアクセスできません
- 契約IDはJWTトークンから自動的に取得され、サーバー側で検証されます

### 5.2 このAPIのスコープ制限

| 項目 | 制限内容 |
|------|---------|
| アクセス可能なデータ | ログイン中のユーザーの契約IDに紐づくユーザー情報のみ |
| 制限方法 | SQL WHERE句: `contract_id = :contractId`（JWTから取得、[入力パラメータ](#21-入力パラメータ詳細)参照） |

### 5.3 JWT構造例

```json
{
  "sub": "usr_001",
  "name": "山田太郎",
  "email": "yamada@example.com",
  "contract_id": "cnt_001",  // ← この契約IDに紐づくデータのみアクセス可能
  "role": "system_admin",
  "exp": 1730390400,
  "iat": 1730386800
}
```

### 5.4 スコープ制限エラー時のレスポンス

他の契約のデータにアクセスしようとした場合、以下のレスポンスが返されます：

**HTTPステータス:** 403 Forbidden

```json
{
  "success": false,
  "error": {
    "code": "PERMISSION_DENIED",
    "message": "この操作を実行する権限がありません",
    "details": {
      "reason": "契約IDが一致しません"
    }
  },
  "timestamp": "2025-11-01T10:00:00+09:00"
}
```

**注:** スコープ制限の詳細は[認証・認可](../../common-specification.md#8-認証認可)を参照してください。

## 6. 備考

- このAPIは[クライアント側キャッシュ戦略](../../common-specification.md#9-クライアントサイドキャッシュ戦略)を採用しています
- サーバー側では契約IDで絞り込んだ全件データを返却します（契約IDはJWTトークンから自動取得）
- 初回アクセス時に全件データを取得し、クライアント側でキャッシュします
- **検索・フィルタリング・ソート・ページングは全てクライアント側で実施してください**
- キャッシュ有効期間: 5分（推奨）
- CRUD操作後は即座にキャッシュを無効化して再取得してください
- データ件数が1,000件を超える場合は、サーバーサイドページネーションへの移行を検討してください
