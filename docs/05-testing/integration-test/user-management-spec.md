## 📋 概要

このドキュメントは、ユーザ管理機能のAPI結合テスト項目を定義します。

### 対象範囲

- APIエンドポイント（Handler層）
- 認証・認可Middleware
- API連携（フロントエンド↔バックエンド）
- 外部サービス連携（Entra ID、メール送信）

### カバレッジ目標

- **全体**: 80%以上
- **認証・認可**: 90%以上
- **エラーハンドリング**: 90%以上

---

## 🎯 テスト方針

### テスト対象

結合テストでは以下をテストします：
- HTTPリクエスト/レスポンス
- ルーティング
- 認証・認可フロー
- バリデーション
- エラーハンドリング
- ミドルウェアチェーン

### テストアプローチ

- 実際のHTTPリクエストをシミュレート
- テストデータベースを使用
- 外部サービス（Entra ID、メール）はモック化
- エンドツーエンドのAPIフロー検証
