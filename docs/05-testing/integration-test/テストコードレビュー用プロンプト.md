# 結合テストコードレビュー用プロンプト

以下の `<機能名>` を対象機能に差し替えて使用してください。

````
以下の手順に従い、<機能名> の結合テストコードをレビューしてください。

## 入力ファイル（必ず読み込むこと）

1. 結合テスト観点表: docs/05-testing/integration-test/integration-test-perspectives.md
2. テストコードレビューチェックリスト: docs/05-testing/integration-test/テストコードレビュー用プロンプト.md（本ファイルのチェックリストセクション）
3. レビュー対象テストコード: tests/integration/<対象機能>/test_<対象ファイル名>_routes.py
4. テスト項目書（存在する場合）: docs/05-testing/integration-test/<対象機能>/テスト項目書_<対象機能>.md

## レビュー手順

1. 入力ファイルをすべて読み込む
2. チェックリストの各項目を順番に確認し、問題があれば該当箇所を特定する
3. 問題が見つかった場合、以下の形式で指摘をまとめ、出力先ファイルに保存する

### 指摘形式

全テスト対象ファイルの指摘を1テーブルにまとめること。

| No | 対象ファイル | 対象テスト / 行 | 指摘内容 | 対応観点 | 修正案 |
|----|-------------|----------------|----------|----------|--------|
| 1  | test_xxx_routes.py | test_yyy / L23 | ... | 4.3.1 | ... |
| 2  | test_xxx_routes.py | test_zzz / L45 | ... | 1.2.3  | ... |

## 出力先

レビュー結果は全テスト対象ファイルをまとめて1ファイルに出力すること。

- `docs/05-testing/integration-test/<対象機能>/temp/review_<対象機能>.md`
- ディレクトリが存在しない場合は作成する
- 指摘が0件の場合も「指摘なし」と記載したファイルを出力する

## 修正対象

指摘が発生した場合、指摘された**全テストファイル**とテスト項目書を修正すること。

- **テストコード（必須）**: 指摘のあったファイルをすべて修正する
  - `tests/integration/<対象機能>/test_<対象ファイル名>_routes.py`
  - その他関連ファイル（conftest.py 等）が存在する場合は同様に対応する
- **テスト項目書（存在する場合は必須）**: `docs/05-testing/integration-test/<対象機能>/テスト項目書_<対象機能>.md`
  - テストコードに追加・変更が生じた場合、テスト項目書のテスト内容・総数・連番も合わせて更新する

## 注意事項

- チェックリストに記載のない問題を発見した場合も指摘すること
- 問題がない項目は「OK」と記録し、すべての項目を確認したことを明示する
````

## テストコードレビューチェックリスト

> 指摘が発生した都度、新しい行を追加してください。


### CL-1: テストコード品質

| No     | チェック項目                                                                                          | 対応観点 |
| ------ | ----------------------------------------------------------------------------------------------------- | -------- |
| CL-1-1 | `unittest.mock.patch` のパスが**テスト対象モジュール内のimport先**を指しているか（import元ではなく）  | -        |
| CL-1-2 | 各テストメソッドのdocstringに観点表に基づいた観点番号と概要が記載されているか                         | -        |
| CL-1-3 | fixture の teardown がロールバック（`db.session.rollback()`）で行われているか                        | -        |

### CL-2: テスト項目書整合性

| No     | チェック項目                                                                       | 対応観点 |
| ------ | ---------------------------------------------------------------------------------- | -------- |
| CL-2-1 | テスト項目書の「テスト総数」とテストコードの実テストメソッド数が一致しているか      | -        |
| CL-2-2 | テスト項目書のNo列に重複・欠番がないか                                              | -        |
| CL-2-3 | テストコードに追加・変更があった場合、テスト項目書の内容・総数・連番も更新されているか | -      |
