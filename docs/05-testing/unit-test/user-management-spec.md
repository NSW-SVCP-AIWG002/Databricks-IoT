## 📋 概要

このドキュメントは、ユーザ管理機能の単体テスト項目を定義します。

### 対象範囲

#### Repository層
- Usersテーブル操作
- Groupsテーブル操作
- Group_Membersテーブル操作

#### UseCase層
- ユーザ検索（getUsersUseCase）
- ユーザ登録（createUserUseCase）
- グループ一覧取得（getGroupsUseCase）

#### UIコンポーネント層
- 検索フォーム
- ユーザ一覧テーブル
- ページネーション
- ユーザ登録モーダル
- 破棄確認ダイアログ
- バリデーション
- データ表示
- アクセシビリティ

### カバレッジ目標

- **全体**: 80%以上
- **Repository層 CRUD操作**: 90%以上
- **UseCase層 認証・権限チェック**: 90%以上
- **UseCase層 ユーザ登録処理**: 90%以上
- **UIコンポーネント層 ユーザー操作**: 90%以上

---

## 🎯 テスト方針

### Repository層

#### テスト対象
- データベースとの直接的なやり取り
- SQL クエリの実行
- データの永続化・取得・更新・削除

#### テストアプローチ
- 実際のテストデータベースを使用（インメモリDBまたはDocker）
- トランザクション単位でロールバック
- 境界値・異常系のテスト

### UseCase層

#### テスト対象
- ビジネスロジックの実装
- バリデーション
- Repository層の呼び出し
- 外部サービス連携（Entra ID、メール送信）

#### テストアプローチ
- Repository層はモック化
- 外部サービス（Graph API、メール送信）はモック化
- ビジネスロジックの正確性を重点的にテスト
- 正常系・異常系の両方をカバー

### UIコンポーネント層

#### テスト対象
- レンダリングの正確性
- ユーザーインタラクション（クリック、入力、フォーカス）
- 状態管理（useState、useReducer）
- バリデーション
- アクセシビリティ

#### テストアプローチ
- React Testing Libraryを使用
- APIはモック化
- ユーザー視点のテスト（実装の詳細には依存しない）
- アクセシビリティの確認
