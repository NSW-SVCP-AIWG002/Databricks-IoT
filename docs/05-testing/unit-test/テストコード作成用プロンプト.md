# 単体テストコード作成用プロンプト
---

````
以下の手順に従い、industry-dashboard の単体テストコードを作成してください。

## 入力ファイル（必ず読み込むこと）

1. UI設計書: docs/03-features/flask-app/industry-dashboard/ui-specification.md
2. 機能設計書: docs/03-features/flask-app/industry-dashboard/workflow-specification.md
3. 単体テスト観点表: docs/05-testing/unit-test/unit-test-perspectives.md
4. 単体テスト実装ガイド: docs/05-testing/unit-test/unit-test-guide.md

※ 上記パスにファイルがない場合は、サブフォルダ（書面レビュー対象資料/ 等）内を探索すること。

## 作業手順

1. UI設計書と機能設計書を読み、機能の振る舞いを把握する
2. 単体テスト観点表を読み、該当する機能に必要な観点を選択する
3. 単体テスト実装ガイドの「テストコード実装例」を参照し、コードの書き方を確認する
4. 設計書に記載された具体的な値（項目名、バリデーション条件、エラーメッセージ等）をテストコードに反映する

## 出力先

tests/unit/test_industry-dashboard.py（責務単位で複数ファイルに分割可）　　

## 出力形式

pytestのテストコードとして出力すること。以下のルールに従う：
- クラスのdocstringに、対応する観点表のセクション番号を記載する
- 各テストメソッドのdocstringに、対応する観点表の項目番号と概要を記載する
- AAAパターン（Arrange-Act-Assert）に従う
- @pytest.mark.unit マーカーを付与する
- 外部依存（DB、API等）はすべてモック化する
- 具体的なコードパターンは単体テスト実装ガイドの「テストコード実装例」を参照する

## 注意事項

- 観点表のすべての項目を機械的に適用しないこと。設計書に該当する機能・項目がある観点のみ選択する
- 設計書に明記されていない挙動は、テストコード内にコメントで「# TODO: 設計書に記載なし、要確認」と記載する
````
