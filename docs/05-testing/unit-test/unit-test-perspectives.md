## 概要

このドキュメントは、Flask SSRアプリケーションの**単体テスト**実装時に確認すべき汎用的なテスト観点を定義します。

**単体テストとは:**
- 個別の関数・メソッドを独立してテスト
- 外部依存（DB、Flask、API）をすべてモック化
- 高速実行（ミリ秒単位）
- 関数の入力と出力のみに焦点

**他のテストとの違い:**

| テスト種類               | 対象                        | ツール                       | 外部依存              |
| ------------------------ | --------------------------- | ---------------------------- | --------------------- |
| **単体テスト（本文書）** | 個別関数・メソッド          | pytest                       | すべてモック化        |
| **結合テスト**           | Flaskルート + DB + ロジック | pytest + Flask.test_client() | DBは実際、APIはモック |
| **E2Eテスト**            | ブラウザ操作                | Selenium / Playwright        | すべて実環境          |

**目的:**
- 関数レベルでのロジック正確性を保証
- リファクタリングの安全性確保
- 高速なフィードバックループ
- コードの可読性・保守性向上

**対象:**
- バリデーション関数
- ユーティリティ関数
- モデルのプロパティ・メソッド（DB非依存）
- ビジネスロジック関数
- エラーハンドリング関数
- セキュリティ関連関数

**対象外（結合テストで実施）:**
- Flaskルート（エンドポイント）
- データベース操作
- 外部API連携
- HTTPリクエスト・レスポンス

**使用方法:**
1. 各関数実装時に、該当するセクションの観点をすべてチェック
2. 観点ごとにテストケースを作成
3. テストがパスするまで実装とテストを繰り返す

**関連ドキュメント:**
- [結合テスト観点表](./integration-test-perspectives.md) - Flaskルート + DBのテスト
- [E2Eテスト観点表](./e2e-test-perspectives.md) - ブラウザ操作のテスト

## テスト環境

**テスト実行環境:**
- Pythonテストフレームワーク: `pytest`
- モック: `pytest-mock` または `unittest.mock`
- 日時固定: `freezegun`

**基本方針:**
- **外部依存をすべてモック化**（DB、Flask、API、ファイルI/O等）
- **純粋な関数のみテスト**
- **テストは独立して実行可能**（他のテストに依存しない）
