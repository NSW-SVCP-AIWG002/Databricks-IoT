## 📋 概要

このドキュメントは、タスク管理アプリケーションのE2Eテスト（End-to-End Test）を実装する際のガイドラインです。ユーザー視点での実際の利用シナリオを検証し、システム全体が正しく動作することを保証します。

**テストケースの書き方、ベストプラクティス、サンプルコードについては [テスト実装ルール](../../00-rules/test-rule.md) を参照してください。**

---

## 📖 このドキュメントの使い方

このガイドは**汎用的なテンプレート・リファレンス**として活用してください。

### 利用フロー

```
e2e-test-guide.md (80項目の汎用チェックリスト)
       ↓ 参照
e2e-test/task-management-spec.md (タスク管理に該当する10項目を詳細化)
       ↓ 実装
e2e/tasks-crud.spec.ts (実際のテストコード)
```

### 具体的な使い方

1. **新機能実装時**: このガイドを開き、E2Eテスト項目一覧を確認
2. **項目選択**: ユーザーフロー、画面操作、エラーハンドリングから該当項目をピックアップ
3. **仕様書作成**: `e2e-test/<機能名>-spec.md` に機能固有のユーザーシナリオを記載
4. **実装ガイド参照**: [テスト実装ルール](../../00-rules/test-rule.md) のサンプルコードを参考に実装
5. **テスト実装**: 実際のE2Eテストコード（`.spec.ts`）を作成

### 注意点

- このガイドは**汎用的なユーザー操作**のみを記載（例: 「ログインできること」）
- **機能固有のシナリオ**は `e2e-test/` に記載（例: 「管理者が契約一覧から特定契約を検索→編集→保存」）

## 🎯 E2Eテストの目的とスコープ

### 目的

- ユーザー視点での実際の利用シナリオが正しく動作することを保証
- クリティカルパス（重要な操作フロー）の動作確認
- ブラウザ互換性の検証
- レスポンシブデザインの動作確認
- システム全体の統合確認

### スコープ

- **対象**: ユーザーが実際に行う操作フロー全体
- **実行環境**: 実ブラウザまたはヘッドレスブラウザ
- **実行速度**: 低速（全体で5分以内）
- **カバレッジ目標**: クリティカルパス100%

### 単体テスト・結合テストとの違い

| 項目 | 単体テスト | 結合テスト | E2Eテスト |
| ------| ---------| ---------| ----------|
| テスト範囲 | 単一の関数・モジュール | 複数のレイヤー | システム全体 |
| 視点 | 開発者視点 | 開発者視点 | ユーザー視点 |
| 操作方法 | 関数呼び出し | API呼び出し | ブラウザ操作 |
| 実行時間 | 高速（< 30秒） | 中速（< 2分） | 低速（< 5分） |
| 目的 | 個別機能の正確性 | 連携の正確性 | 全体フローの正確性 |

## 🛠️ テスト環境・ツール

### Playwright

- **ブラウザ**: Chromium, Firefox, WebKit（Safari）をサポート
- **ヘッドレスモード**: CI/CDで高速実行
- **UIモード**: デバッグ時に視覚的に確認
- **スクリーンショット**: 失敗時に自動撮影
- **ビデオ録画**: テスト実行の様子を記録

### 設定ファイル

```typescript
// playwright.config.ts
export default defineConfig({
  testDir: './e2e',
  use: {
    baseURL: 'http://localhost:3001',
    trace: 'on-first-retry',
  },
  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } },
    { name: 'firefox', use: { ...devices['Desktop Firefox'] } },
    { name: 'webkit', use: { ...devices['Desktop Safari'] } },
  ],
})
```
