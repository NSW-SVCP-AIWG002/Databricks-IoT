## 概要

このドキュメントは、Flask SSRアプリケーションの**E2Eテスト（End-to-End Test）**実装時に確認すべき汎用的なテスト観点を定義します。

**E2Eテストとは:**
- エンドユーザー視点で**業務フローが最初から最後まで成立するか**を確認するテスト
- 実際のブラウザで実行
- UI操作（クリック、入力等）を手動テストまたは自動化
- 個別ロジックの検証ではなく、業務シナリオの成立を検証

**他のテストとの役割分担:**

| テスト種類              | 対象                        | 確認する内容                                   |
| ----------------------- | --------------------------- | ---------------------------------------------- |
| **単体テスト**          | 個別関数・メソッド          | ロジックの正確性（True/False、例外スロー）     |
| **結合テスト**          | Flaskルート + DB + ロジック | HTTPステータス、DB反映、メッセージ表示         |
| **E2Eテスト（本文書）** | ブラウザ操作                | **業務フローが完了するか、画面状態が正しいか** |

**E2Eテストで確認しないこと（単体・結合で検証済み）:**
- バリデーションロジックの詳細（種別ごとのエラー検証）
- DB格納値の詳細（初期値、自動採番、タイムスタンプ等）
- HTTPステータスコードの詳細
- CSV生成ロジック（エスケープ・エンコーディング）
- ログ出力内容
- セキュリティ攻撃パターン（SQLインジェクション、XSS等）

**使用方法:**
1. 実装完了後、各業務フローのテストシナリオを作成
2. シナリオに沿ってブラウザ操作を自動化
3. テストがパスするまで実装を修正

## テスト環境

**テスト実行環境:**
- ブラウザ自動化ツール: `Selenium` または `Playwright`
- ブラウザ: Chrome最新版
- テストフレームワーク: `pytest` + `pytest-selenium` または `pytest-playwright`

**テスト環境:**
- 本番に近い環境（ステージング環境推奨）
- 外部API（Databricks）も実際に接続
- テスト用データベース（本番とは分離）

## 1. ログインフロー

| No  | 観点                   | テストケース               | 期待値                           |
| --- | ---------------------- | -------------------------- | -------------------------------- |
| 1.1 | 正常ログイン           | 有効な認証情報でログイン   | ダッシュボード画面へ遷移する     |
| 1.2 | ログイン失敗           | 無効な認証情報でログイン   | エラーメッセージが表示される     |
| 1.3 | セッション維持         | ログイン後、別ページへ遷移 | ログイン状態が維持される         |
| 1.4 | セッションタイムアウト | 長時間放置後にアクセス     | ログイン画面へリダイレクトされる |

## 2. 検索・一覧表示フロー

| No   | 観点                       | テストケース                         | 期待値                                                           |
| ---- | -------------------------- | ------------------------------------ | ---------------------------------------------------------------- |
| 2.1  | 一覧初期表示               | 一覧画面にアクセス                   | データ一覧が表示される                                           |

| 2.4  | 検索結果0件                | ヒットしない条件で検索               | 一覧のヘッダーのみ表示される（一覧要素、件数非表示）             |
| 2.5  | 検索結果件数表示           | 検索実行後                           | 検索結果件数が表示される                                         |
| 2.6  | 現在の検索条件表示         | 検索条件指定/未指定で検索            | 指定した条件が表示される（未指定時は「なし」）                   |
| 2.7  | ページ移動                 | ページ番号クリック                   | 検索条件を保持して該当ページのデータが表示される                 |
| 2.8  | ページングアクティブ状態   | ページ移動後                         | 現在ページがアクティブ表示され、±1のページ番号が表示される       |
| 2.9  | 部分ソート切り替え         | テーブルヘッダーを繰り返しクリック   | 昇順→降順→デフォルトの順で切り替わり、アイコンが変化する         |
| 2.10 | 部分ソート別カラムリセット | ソート中に別カラムのヘッダークリック | 押下カラムのみソート対象となり、他カラムのソートがリセットされる |
