## 📋 概要

このドキュメントは、ユーザ管理機能のE2Eテスト項目を定義します。

### 対象範囲

#### ユーザ一覧画面（SCR007-user-management）
- ユーザ一覧画面の表示
- 検索・フィルタリング機能
- ソート機能
- ページング機能
- 詳細画面への遷移

#### ユーザ登録フロー
- ユーザ登録モーダルの表示・操作
- フォーム入力とバリデーション
- ユーザ登録処理
- 成功・エラー時の挙動

### カバレッジ目標

- **主要ユーザーフロー**: 100%
- **エラーハンドリング**: 90%以上

---

## 🎯 テスト方針

### テスト対象

E2Eテストでは以下をテストします：
- ユーザーの実際の操作フロー
- ブラウザでの画面表示
- フロントエンドとバックエンドの統合
- 画面遷移
- エラー表示
- フォームバリデーション
- サーバーとの連携

### テストアプローチ

- Playwrightを使用
- 実際のブラウザで実行
- テストデータベースを使用
- 外部サービス（Entra ID、メール）はモック化
- ユーザー視点での操作シナリオ
