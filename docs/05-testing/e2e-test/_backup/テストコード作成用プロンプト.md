````
以下の手順に従い、<機能名> のE2Eテストコード（Playwright）を作成してください。

## 入力ファイル（必ず読み込むこと）

1. UI設計書: docs/03-features/flask-app/<機能名>/ui-specification.md
2. 機能設計書: docs/03-features/flask-app/<機能名>/workflow-specification.md
3. E2Eテスト観点表: docs/05-testing/e2e-test/e2e-test-perspectives.md
4. E2Eテスト実装ガイド: docs/05-testing/e2e-test/e2e-test-guide.md

※ 上記パスにファイルがない場合は、サブフォルダ（書面レビュー対象資料/ 等）内を探索すること。

## 作業手順

1. UI設計書と機能設計書を読み、ユーザーの操作フロー（CUJ: Critical User Journey）を把握する
2. E2Eテスト観点表を読み、該当する機能に必要な観点を選択する
3. E2Eテスト実装ガイドの「テスト実施例」を参照し、コードの書き方を確認する
4. 設計書に記載された具体的な値（画面名、ボタン名、遷移先、表示メッセージ等）をテストコードに反映する

## 出力先

tests/e2e/test_<機能名>_flow.py

## 出力形式

pytest + Playwrightのテストコードとして出力すること。以下のルールに従う：
- クラスのdocstringに、対応する観点表のセクション番号を記載する
- 各テストメソッドのdocstringに、対応する観点表の項目番号と概要を記載する
- ユーザー操作フロー単位でテストを構成する
- バリデーション詳細や境界値検証は単体テスト・結合テストで担保済みのため、業務フローの完走性に集中する
- 具体的なコードパターンはE2Eテスト実装ガイドの「テスト実施例」を参照する

## 注意事項

- 観点表のすべての項目を機械的に適用しないこと。設計書に該当する機能・項目がある観点のみ選択する
- 設計書に明記されていない挙動は、テストコード内にコメントで「# TODO: 設計書に記載なし、要確認」と記載する
````
