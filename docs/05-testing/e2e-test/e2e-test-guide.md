## 📋 概要

このドキュメントは、Flask + Jinja2 アプリケーションのE2Eテスト（End-to-End Test）を実施する際のガイドラインです。

`docs/03-features/flask-app/xxxx/`配下の各機能の設計書と[E2Eテスト観点](./e2e-test-perspectives.md)を参照し、手動テストを実施します。必要に応じて Playwright でテストコードを実装します。

---

## 📖 このドキュメントの使い方

このガイドは**テスト実施のリファレンス**として活用してください。

### 利用フロー

```
各機能の設計書（振る舞い定義）
       ↓ 参照
e2e-test-perspectives.md (汎用的な観点チェックリスト)
       ↓ 参照・選択
e2e-test-guide.md (テスト実施例)
       ↓ 実施
手動テスト または tests/e2e/test_<機能名>_flow.py (自動化時)
```

### 具体的な使い方

1. **機能設計書確認**: `docs/03-features/flask-app/xxxx/`配下のUI設計書(ui-specification.md)、機能設計書(workflow-specification.md)を読み、ユーザーの操作フロー（CUJ）を確認
2. **観点確認**: [E2Eテスト観点](./e2e-test-perspectives.md) を開き、関連する観点を確認
3. **項目選択**: チェックリストから該当する機能に必要な項目をピックアップ
4. **テスト実施例確認**: このガイドの「テスト実施例」を参照し、テストの進め方を確認
5. **テスト実施**: 観点に基づき手動テストを実施。自動化する場合はテストコード（`test_<機能名>_flow.py`）を作成

### 注意点

- [E2Eテスト観点](./e2e-test-perspectives.md) は**汎用的な観点**を網羅的に記載
- このガイドは**テスト実施例**を提供（手動テスト手順、自動化時のコードパターン）
- **テストコードの実装ルール**は [テスト実装ルール](../../00-rules/test-rule.md) を参照

## 🎯 E2Eテストの目的とスコープ

### 目的

- ユーザー視点での実際の利用シナリオが正しく動作することを保証
- クリティカルパス（重要な操作フロー）の動作確認
- ブラウザ互換性の検証
- レスポンシブデザインの動作確認

### スコープ

- **対象**: ユーザーが実際に行う操作フロー全体
- **実行方式**: 手動テスト基本、必要時にPlaywrightで自動化
- **カバレッジ目標**: クリティカルパス100%

### 単体テスト・結合テストとの違い

| 項目       | 単体テスト             | 結合テスト     | E2Eテスト          |
| ---------- | ---------------------- | -------------- | ------------------ |
| テスト範囲 | 単一の関数・モジュール | 複数のレイヤー | システム全体       |
| 視点       | 開発者視点             | 開発者視点     | ユーザー視点       |
| 操作方法   | 関数呼び出し           | API呼び出し    | ブラウザ操作       |
| 目的       | 個別機能の正確性       | 連携の正確性   | 全体フローの正確性 |

## 📊 テスト実施例

このセクションでは、[E2Eテスト観点](./e2e-test-perspectives.md) の項目を、どのようにテスト実施するかの例を示します。

### 手動テストの実施方法

観点表の各項目に対して、ブラウザ上で操作を行い結果を確認します。

### 1. 検索・一覧表示フローのテスト例

**手動テスト手順（観点: 2. 検索・一覧表示フロー）:**

1. 一覧画面にアクセスし、データ一覧が表示されることを確認（2.1）
2. ▼ボタンをクリックし、検索フォームが開くことを確認（2.2）
3. 検索条件を入力→検索ボタンクリック→条件に合致するデータが表示されることを確認（2.3）
4. 検索結果件数が正しく表示されることを確認（2.5）
5. ページ番号をクリックし、該当ページのデータが表示されることを確認（2.7）

**自動化する場合（Playwright）:**

```python
# tests/e2e/test_device_management_flow.py
# 観点: e2e-test-perspectives.md > 2. 検索・一覧表示フロー

import pytest
from playwright.sync_api import Page

class TestDeviceSearchFlow:
    """デバイス管理 - 検索・一覧表示フロー
    観点: 2.1 一覧初期表示, 2.3 検索実行, 2.5 検索結果件数表示
    """

    def test_search_and_display(self, page: Page):
        """2.3: 検索条件入力→検索ボタンクリック→条件に合致するデータが表示される"""
        # Arrange
        page.goto("/devices")

        # Act
        page.click(".search-toggle")
        page.fill("#search-device-id", "DEV-001")
        page.click("#search-button")

        # Assert
        assert page.locator(".device-row").count() > 0
        assert page.locator(".result-count").inner_text() != "0"
```

### 2. 登録フローのテスト例

**手動テスト手順（観点: 3. 登録フロー）:**

1. 登録ボタンをクリックし、登録用モーダルが表示されることを確認
2. 必要項目を入力し、送信ボタンをクリック
3. 成功メッセージが表示され、一覧に新規データが表示されることを確認（3.1）

**※ 上記は実施例です。実際のテストでは、[E2Eテスト観点](./e2e-test-perspectives.md) から該当する機能の観点を選択し、同様の手順でテストを実施してください。**
