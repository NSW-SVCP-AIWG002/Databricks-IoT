# フロントエンドアーキテクチャ

## 概要

Databricks IoTシステムのフロントエンドは、Flask + Jinja2テンプレートエンジンをベースとしたサーバーサイドレンダリング方式を採用しています。HTML + CSSのクラシックな構成により、シンプルで保守性の高い設計を実現します。

## 技術スタック

- **Webフレームワーク**: Flask (Python)
- **テンプレートエンジン**: Jinja2
- **HTML**: HTML5 (セマンティックHTML)
- **スタイリング**: CSS3 (シンプルなカスタムCSS)
- **JavaScript**: 最小限の使用 (フォーム検証、UI補助)
- **埋め込みコンテンツ**: Databricksダッシュボード・対話型AI (iframe)
- **認証**: Databricks User認証 + リバースプロキシヘッダ

## 主要な設計原則

1. **サーバーサイドレンダリング (SSR)**
   - Jinja2によるサーバーサイドテンプレート生成
   - 初期ロードの高速化
   - シンプルなアーキテクチャ

2. **MVCパターン**
   - Model: データベースアクセス・ビジネスロジック
   - View: ビュー関数によるリクエスト処理
   - Template: Jinja2テンプレートによるUI生成
   - 明確な責務の分離

3. **シンプリシティ優先**
   - 複雑なJSフレームワークを使用しない
   - HTML + CSSのクラシックな構成
   - 保守性・可読性の向上
   - Databricksへのデプロイの容易性

4. **Databricks統合**
   - Databricksダッシュボードのiframe埋め込み
   - 対話型AIパネルの共有
   - Databricks User認証との連携

## アーキテクチャレイヤー

### 1. ルーティング層 (Flask Blueprint)
- **Blueprint構成**:
  - `dashboard_bp`: ダッシュボードルート (/ ※ホーム画面兼用, /dashboard)
  - `admin_bp`: 管理機能ルート (/admin/*)
  - `alert_bp`: アラート機能ルート (/alert/*)
  - `notice_bp`: 通知機能ルート (/notice/*)
  - `account_bp`: アカウントルート (/account/*)
  - `transfer_bp`: インポートルート (/transfer/csv-import)
- URLルーティング定義
- ミドルウェア (リバースプロキシヘッダからのユーザー識別)
- **認証**: Databricks標準機能（アプリでは実装しない）

### 2. ビュー層 (View Functions)
- **ダッシュボードビュー**:
  - Databricksダッシュボード埋め込み表示（ホーム画面）
  - 最新アラート情報表示
  - 対話型AI (ダッシュボード内)
- **管理画面ビュー**:
  - デバイス管理 (CRUD操作)
  - ユーザー管理 (CRUD操作)
  - 組織管理 (CRUD操作)
  - デバイス台帳管理 (CRUD操作)
- **アラート機能ビュー**:
  - アラート設定管理 (CRUD操作)
  - アラート履歴表示
- **通知機能ビュー**:
  - メール通知設定管理 (CRUD操作)
  - メール通知履歴表示
- **アカウントビュー**:
  - 言語設定
  - ユーザ情報参照
  - **注**: パスワード管理はEntra ID側で行うため、パスワード再設定機能は実装しない
- **インポート/エクスポートビュー**:
  - CSVインポート
  - CSVエクスポート
- リクエスト処理とレスポンス生成
- データベースアクセス (SQL Warehouse、OLTP)

### 3. テンプレート層 (Jinja2)
- **ベーステンプレート**:
  - base.html: 共通レイアウト (ヘッダー、フッター、ナビゲーション)
  - テンプレート継承とブロック定義
- **管理画面テンプレート**:
  - フォーム
  - テーブル表示 (データ一覧)
- **ダッシュボードテンプレート**:
  - iframe埋め込み (Databricksダッシュボード)
- テンプレートフィルター・マクロの活用

### 4. 静的ファイル層
- **CSS**:
  - main.css: 共通スタイル、レイアウト、タイポグラフィ
  - admin.css: 管理機能専用スタイル
  - alert.css: アラート機能専用スタイル
  - notice.css: 通知機能専用スタイル
  - dashboard.css: ダッシュボード専用スタイル
  - account.css: アカウント専用スタイル
  - transfer.css: インポート/エクスポート専用スタイル
- **JavaScript** (最小限):
  - common.js: 共通処理
  - form-validation.js: クライアントサイドバリデーション
- **Images**: ロゴ、アイコン等

## パフォーマンス最適化

- **静的ファイル配信**: Flaskの静的ファイル配信最適化
- **テンプレートキャッシュ**: Jinja2のテンプレートコンパイルキャッシュ
- **CSS/JSの最小化**: 本番環境での圧縮
- **画像最適化**: 適切なサイズとフォーマット
- **ブラウザキャッシュ**: 静的リソースのCache-Control設定
- **Gzip圧縮**: HTMLレスポンスの圧縮
- **データベースクエリ最適化**: SQL Warehouse/OLTP接続の効率化
- **iframe遅延読み込み**: Databricksダッシュボードの遅延ロード

## セキュリティ対策

- **認証基盤**:
  - **Databricks User認証**: Databricks標準機能（アプリでは実装しない）
  - **Entra ID (Azure Active Directory)**: ユーザ認証・トークン発行の中核
  - リバースプロキシによるユーザ識別子・アクセストークンの付与
  - FlaskアプリではリバースプロキシヘッダからユーザーIDを取得
  - **アクセス元IP制限**: Databricks Apps公開フロントへのIP制限
- **CSRF対策**:
  - Flask-WTF による CSRFトークン生成・検証
  - フォーム送信時のトークン検証
- **入力バリデーション**:
  - サーバーサイドでの厳密な検証
  - SQLインジェクション対策
  - XSS対策 (Jinja2の自動エスケープ)
- **セキュアヘッダ**:
  - Content-Security-Policy
  - X-Frame-Options (iframe制御)
  - X-Content-Type-Options
- **データアクセス制御**: ユーザー権限に基づいたデータ範囲の制限
