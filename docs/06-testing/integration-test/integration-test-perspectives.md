## 概要

このドキュメントは、Flask SSRアプリケーションの**結合テスト**実装時に確認すべき汎用的なテスト観点を定義します。

**結合テストとは:**
- 複数コンポーネント（Flask + DB + ロジック）を統合してテスト
- Flask test_clientを使用してHTTPリクエストを送信
- テスト用DBで実際のSQL操作を実行
- 外部API（Databricks）は原則モック化（例外: Unity Catalog クエリは 6.3 にて実接続で検証）
- HTMLレンダリング結果を検証

**他のテストとの違い:**

| テスト種類               | 対象                        | ツール                       | 外部依存              |
| ------------------------ | --------------------------- | ---------------------------- | --------------------- |
| **単体テスト**           | 個別関数・メソッド          | pytest                       | すべてモック化        |
| **結合テスト（本文書）** | Flaskルート + DB + ロジック | pytest + Flask.test_client() | DBは実際、APIはモック |
| **E2Eテスト**            | ブラウザ操作                | Selenium / Playwright        | すべて実環境          |

**目的:**
- 実装時に漏れなくテストケースを作成するためのチェックリスト
- 品質担保のための共通基準
- テストコード作成の効率化

**対象:**
- Flask SSRルート（エンドポイント）
- ビジネスロジック
- データベース操作（実際のDB使用）
- 外部API連携（モック化）
- HTMLレンダリング
- HTTPレスポンス

**使用方法:**
1. 各機能実装時に、該当するセクションの観点をすべてチェック
2. 観点ごとにテストケースを作成
3. テストがパスするまで実装とテストを繰り返す

**関連ドキュメント:**
- [単体テスト観点表](./unit-test-perspectives.md) - 個別関数のテスト
- [E2Eテスト観点表](./e2e-test-perspectives.md) - ブラウザ操作のテスト

## テスト環境

**テスト実行環境:**
- Pythonテストフレームワーク: `pytest`
- Flaskテスト: `Flask.test_client()`
- データベース: テスト用DB（本番とは分離）
- 外部API: モック化

**推奨テストライブラリ:**
- `pytest`: テストフレームワーク
- `pytest-flask`: Flask統合
- `pytest-mock`: モック機能
- `factory_boy`: テストデータ生成
- `freezegun`: 日時固定
- `responses`: HTTPモック

### モックの使用方針

**Databricks SCIM API はモック化:**
- ユーザー・グループ管理のSCIM API呼び出しはモック化

**Unity Catalog クエリは実接続で確認（Section 6.3）:**
- `DEV_DATABRICKS_TOKEN` が設定された開発環境で実際にクエリを実行
- `@pytest.mark.databricks` を付与し、`pytest -m databricks` で単独実行可能
- 接続不要な環境では `pytest -m "not databricks"` でスキップ可能

**データベース（MySQL）はモック化しない:**
- テスト用DBを使用（トランザクションロールバック）
- 実際のSQLクエリを実行してテスト
