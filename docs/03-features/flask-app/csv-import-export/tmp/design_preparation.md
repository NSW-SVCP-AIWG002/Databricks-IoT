## 把握した要件

### 機能要件（FR-004-8）

#### FR-004-8-1: CSVエクスポート

**出典:** `functional-requirements.md` 427-435行目

**要件概要:**
- 各マスタの一覧画面にエクスポートボタンを配置（個別のエクスポート画面は存在しない）
- エクスポート対象: 一覧画面の絞り込み条件が適用された状態のデータ
- 対象データ: ユーザー、組織、デバイス、アラート設定などのマスタデータ（IoTセンサーデータは対象外）
- CSV形式: UTF-8 BOM付き、カンマ区切り
- 出力項目: 各マスタの全項目を出力
- ファイル名形式: `{データ種別}_{YYYYMMDD_HHmmss}.csv`

**詳細仕様:**
- エクスポートボタンは各マスタの一覧画面に配置
- 一覧画面で適用されている絞り込み条件（検索条件、ソート条件）がそのまま反映される
- 権限: 各マスタのC（登録）/U（更新）権限に従う

**対象マスタ:**
1. デバイス（devices）
2. ユーザー（users）
3. 組織（organizations）
4. アラート設定（alert_settings）
5. デバイス台帳（device_inventory）
6. メール通知設定（mail_settings）

#### FR-004-8-2: CSVインポート

**出典:** `functional-requirements.md` 436-450行目

**要件概要:**
- 専用のCSVインポート画面（TRF-001）
- 対象マスタ選択: プルダウンで対象マスタ（ユーザー、組織、デバイス、アラート設定、デバイス台帳、メール通知設定）を選択
- 対象データ: ユーザー、組織、デバイス、アラート設定などのマスタデータ（IoTセンサーデータは対象外）
- インポート機能:
  - CSVファイルのアップロード
  - データの一括登録・更新
  - エラーチェック（形式チェック、重複チェック、必須項目チェック）
  - エラー時はロールバック
- ファイル形式: CSV形式（UTF-8 BOM付き、カンマ区切り）
- 権限: システム保守者、管理者、販社ユーザのみアクセス可能
  - サービス利用者は権限なし（グローバルメニューで非表示）
  - その他は各マスタのC（登録）/U（更新）権限に従う（プルダウン上への表示で制御）

**画面情報:**
- 画面ID: TRF-001
- 画面名: CSVインポート画面
- スラッグ: `/transfer/csv-import`
- 概要: マスタデータの一括インポート

### 非機能要件

#### NFR-SEC-001: 認証・認可

**出典:** `non-functional-requirements.md` 195-240行目

**要件:**
- Databricks User認証 + Entra ID（Flaskアプリでは実装しない）
- リバースプロキシヘッダからユーザー情報を取得してユーザー識別
- ロールベースアクセス制御（RBAC）の実装

**ロール定義:**
| ロール | 権限レベル | CSVインポートアクセス |
|--------|-----------|---------------------|
| システム保守者 | 最上位権限（保守作業含む） | ◎ 制限なく利用可能 |
| 管理者 | 管理機能全般へのアクセス可能 | ◎ 制限なく利用可能 |
| 販社ユーザ | 自社データに関連する機能のみアクセス可能 | ○ 制限下で利用可 |
| サービス利用者 | 参照機能とアラート設定のみアクセス可能 | - 利用不可 |

#### NFR-SEC-006: ロールベースアクセス制御要件

**出典:** `non-functional-requirements.md` 339-376行目

**要件:**
- 機能レベルでロールに応じたアクセス制御を実装
- 権限のないユーザーが機能にアクセスできないようにする

**主要な権限制限:**
- **CSVインポート**: システム保守者、管理者、販社ユーザのみアクセス可能
- グローバルメニューでの表示制御: サービス利用者には非表示

#### NFR-SEC-007: データスコープ制限要件

- サービス利用者は自社データのみアクセス可能

**データスコープ:**
| ロール | データアクセス範囲 |

| 管理者 | 全データにアクセス可能 |

### 技術要件

#### TR-BE-004: CSVインポート/エクスポート機能

**出典:** `technical-requirements.md` 264-493行目

**必須ライブラリ:**
```python
pandas==2.0+  # CSV処理
chardet==5.0+  # 文字コード検出
```

**CSVエクスポート実装:**
- 実装方式: 各マスタの一覧APIに `?export=csv` パラメータを追加することでCSVレスポンスを返却
- エンドポイント例:
  - `GET /api/admin/devices?export=csv` - デバイス一覧のCSVエクスポート
  - `GET /api/admin/users?export=csv` - ユーザー一覧のCSVエクスポート
  - `GET /api/admin/organizations?export=csv` - 組織一覧のCSVエクスポート
- 権限チェック: 各マスタのC/U権限に従う（一覧APIと同じ権限チェックロジック）

**CSVインポート実装:**
- マスタ種別ごとの設定（MASTER_CONFIGS）:
  - devices: デバイス、必須カラム: デバイスID、デバイス名、組織ID、販社ID
  - users: ユーザー、必須カラム: ユーザーID、ユーザー名、メールアドレス、ロール
  - organizations: 組織、必須カラム: 組織ID、組織名、販社ID
- 文字コード検出: chardetライブラリで自動検出
- エラーチェック:
  - ファイルサイズチェック（10MB制限）
  - 必須カラムチェック
  - 形式チェック（CSV解析エラー）
  - 重複チェック
  - 必須項目チェック
- エラー時はロールバック

**ルーティング実装:**
- Blueprint: `transfer_bp`
- エンドポイント:
  - `GET /transfer/csv-import`: CSVインポート画面表示
  - `POST /transfer/api/import`: CSVインポートAPI
- 権限制御: `@role_required(Role.SYSTEM_ADMIN, Role.ADMIN, Role.SALES_USER)`

**制約事項:**
- ファイルサイズ制限: 10MB以下（App Compute制約）
- 対応文字コード: UTF-8（BOM付き/なし）、Shift-JIS、EUC-JP（自動検出）
- CSV形式: カンマ区切り、ヘッダー行必須
- スコープ制限: インポート/エクスポートともにロール別データスコープを適用

### 要件定義書に記載がない項目（設計で補完）

#### 1. CSVテンプレートダウンロード機能

**補完理由:**
- ユーザーがCSVフォーマットを理解しやすくするため
- 初めてインポートする際のサンプルとして有用

**設計での追加:**
- CSVインポート画面に「テンプレートダウンロード」ボタンを配置
- マスタ種別ごとのCSVテンプレートファイルを生成・ダウンロード

#### 2. インポート結果サマリー表示

**補完理由:**
- インポート処理の成功/失敗を明確にユーザーに伝えるため
- 登録件数、更新件数、エラー件数を表示

**設計での追加:**
- インポート成功時: 「登録: N件、更新: M件、合計: N+M件のデータをインポートしました」
- インポート失敗時: 「エラーが発生しました。以下の行を修正してください」+ エラー一覧

#### 3. ファイルアップロード前のプレビュー機能

**補完理由:**
- ユーザーがファイルをアップロードする前に内容を確認できるようにするため
- 誤ったファイルのアップロードを防止

**設計での追加:**
- ファイル選択後、最初の10行をテーブル形式でプレビュー表示
- プレビュー確認後に「インポート実行」ボタンを押下

#### 4. キャンセル機能

**補完理由:**
- 処理中にユーザーがインポートをキャンセルできるようにするため
- 大量データのインポート中に誤ってアップロードした場合の対応

**設計での追加:**
- 処理中に「キャンセル」ボタンを表示
- キャンセル時はロールバック処理を実行

## 設計方針

### 1. Flask SSR（Server-Side Rendering）アプローチ

**方針:**
- JSONレスポンスではなく、HTMLレスポンス（`render_template()`）またはリダイレクト（`redirect()`）を返す
- ルートは `/api/transfer/csv-import` ではなく `/transfer/csv-import` の形式
- クライアントサイドJavaScriptは最小限（Jinja2テンプレートでサーバーサイドレンダリング）

**適用例:**
- `GET /transfer/csv-import`: CSVインポート画面表示（HTML）
- `POST /transfer/api/import`: CSVインポート実行（リダイレクト → 成功時: 画面再表示、失敗時: エラーメッセージ付きフォーム再表示）

### 2. セキュリティファースト設計

**方針:**
- すべてのルートで権限チェックを実装（`@role_required` デコレーター）
- データスコープ制限をアプリケーション層で実装（WHERE句による自動フィルタリング）
- サーバーサイドバリデーションを必須実施（WTForms）
- SQLインジェクション対策（SQLAlchemy ORMのプリペアドステートメント）
- CSRF対策（Flask-WTF）

**適用例:**
- CSVインポート画面: `@role_required(Role.SYSTEM_ADMIN, Role.ADMIN, Role.SALES_USER)`
- CSVエクスポート: 各マスタのC/U権限チェック + データスコープ制限

### 3. エラーハンドリング重視

**方針:**
- エラー時は必ずロールバック（トランザクション管理）
- エラーメッセージはユーザーフレンドリーに表示（行番号、カラム名、エラー内容）
- エラーログを詳細に記録（監査証跡）

**適用例:**
- CSVインポート失敗時: データベースをロールバック、エラー一覧を画面に表示
- CSVエクスポート失敗時: エラーページ表示

### 4. パフォーマンス最適化

**方針:**
- ファイルサイズ制限（10MB）を厳守
- 大量データ処理時はプログレスバー表示（将来実装）
- データベースクエリ最適化（インデックス活用、N+1問題回避）

**適用例:**
- CSVインポート: pandasによる効率的なCSV処理
- CSVエクスポート: 一覧APIと同じクエリを使用（インデックス活用）

### 5. ユーザビリティ重視

**方針:**
- CSVテンプレートダウンロード機能を提供
- ファイルアップロード前のプレビュー機能を実装
- インポート結果サマリーを分かりやすく表示

**適用例:**
- CSVテンプレートボタン: マスタ種別ごとのサンプルCSVをダウンロード
- プレビュー表示: 最初の10行をテーブル形式で表示
