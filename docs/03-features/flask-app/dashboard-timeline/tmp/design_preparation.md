# ダッシュボードタイムライングラフ - 設計準備書

## 1. 要件分析

### 1.1 機能要件（FR-006より）

FR-006-1 ダッシュボード表示の要件のうち、タイムライングラフに関連する要件:

| 要件 | 内容 | タイムラインでの対応 |
|------|------|---------------------|
| リアルタイムデータ表示 | センサーデータの可視化 | Silver層から指定期間のセンサーデータを折れ線グラフで表示 |
| 時系列グラフ | 期間指定による絞り込み | 開始・終了日時ピッカーで期間指定 |
| データアクセス制御 | ユーザーロールに応じた制限 | sensor_data_view経由でRow Access Policy適用 |
| 全ロールアクセス可能 | 全4ロールがダッシュボードにアクセス可能 | 同様に全ロールアクセス可能 |

### 1.2 ユーザー指定要件

| 項目 | 仕様 |
|------|------|
| グラフ種別 | ECharts 折れ線グラフ（Line Chart） |
| 凡例（レジェンド） | 2つ選択可能（左Y軸・右Y軸に各1つのセンサー項目を割り当て） |
| Y軸目盛り分割 | 10分割（ECharts splitNumber: 10） |
| データプロット | 全データ点をプロット（間引きなし） |
| データソース | Silver層 `silver_sensor_data`（集約なし、生データ直接参照） |
| 最小値/最大値 | ユーザー指定可能。ECharts min/maxで範囲外データを描画から除外 |
| 期間指定 | 開始・終了日時ピッカー |
| CSV出力 | あり |
| デバイスモード | 指定デバイス / ツリー連動 |

### 2.2 最も類似するパターン

- **データ取得方式**: 表ガジェット(DTB)と類似（Silver層直接、日時範囲指定、集約なし）
- **グラフ描画方式**: 棒グラフ(DBG)と類似（ECharts、凡例選択、最小値/最大値設定）
- **ガジェット登録方式**: 棒グラフ(DBG)と類似（指定デバイス/ツリー連動、センサー選択、部品サイズ）

|------|-----|------|

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| DTL-001 | タイムライングラフ表示 | 指定期間のセンサーデータを折れ線グラフで表示 |

| DTL-004 | ガジェット登録 | タイムライングラフをダッシュボードに登録 |

## 5. 新規セクション・追加項目の根拠

### 5.1 UI仕様書に追加するセクション

| 追加項目 | 根拠 |
|---------|------|
| Dual Y-Axis凡例選択仕様 | 既存ガジェットは単一Y軸のため、左右Y軸への項目割り当てUI仕様が新規必要 |
| 最小値/最大値設定（2軸分） | 棒グラフは1軸分のmin/maxのみ。タイムラインは左右2軸それぞれの設定が必要 |
| ECharts Line Chart描画仕様 | 既存はbar/pie/stacked barのみ。折れ線グラフ固有の描画設定（線スタイル、マーカー等）を定義 |

### 5.2 ワークフロー仕様書に追加するセクション

| 追加項目 | 根拠 |
|---------|------|
| 2系列データ整形処理 | 1つのAPIレスポンスで2つのY軸系列のデータを返却する必要がある |
| ECharts min/maxフィルタリング仕様 | サーバー側でデータを返却しつつ、ECharts側でmin/max範囲外データを描画除外する方式の定義 |

### 6.2 データ取得制約

| 制約 | 値 | 根拠 |
|------|-----|------|
| 最大取得範囲 | 24時間 | 表ガジェット(DTB)と同様。Silver層の生データ（10秒間隔）のため大量データ抑制 |
| 最大取得件数 | 8,640件 | 24時間 × 60分 × 6回/分 = 8,640件（10秒間隔） |
| データ間隔 | 約10秒 | Silver層の生データ間隔 |
