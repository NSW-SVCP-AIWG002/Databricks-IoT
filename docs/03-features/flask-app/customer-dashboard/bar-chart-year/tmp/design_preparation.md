# ダッシュボード棒グラフ（年）機能 - 設計準備書

## 1. 要件理解

### 1.1 機能概要
ダッシュボード棒グラフ（年）機能は、IoTデバイスから収集したセンサーデータを棒グラフで可視化する機能である。選択した年から直近3年のゴールド層（月次サマリ）データを表示する。

### 1.2 関連要件（FR-006 分析機能）
| 要件ID | 要件内容 | 本機能での対応 |
|--------|----------|----------------|
| FR-006-001 | ダッシュボード表示 | 棒グラフによるデータ可視化 |
| FR-006-002 | 分析期間指定 | 日時ピッカーによる期間選択 |

### 1.3 非機能要件
| 要件 | 内容 | 対応方針 |
|------|------|----------|
| 性能 | ページ表示3秒以内、API応答500ms以内 | クエリ最適化、適切なインデックス利用 |
| セキュリティ | ロールベースアクセス制御 | 組織閉包テーブルによるデータスコープ制限 |
| ユーザビリティ | 直感的な操作 | シンプルなUI設計 |

## 3. 画面分析（bar-chart-year.png）

### 3.1 画面構成要素
1. **タイトル部**
   - 画面タイトル：「棒グラフ（年）」
   - サブタイトル/ラベル：「test」（デバイス名または設定名）

2. **操作部**
   - CSV出力ボタン
   - 年選択：ドロップダウン
   - 基準月選択：ドロップダウン

3. **グラフ表示部**
   - 縦軸：値（単位付き）
   - 横軸：時間軸（表示単位に応じた目盛り）
   - 凡例：年月

### 3.2 画面操作フロー
1. デバイス選択（事前に設定済み）
2. 年選択
3. 基準月選択
4. グラフ表示更新
5. （オプション）CSV出力

## 4. 設計上の考慮事項

### 4.1 セキュリティ考慮
- ユーザーの組織IDに基づくデータフィルタリング
- 組織閉包テーブルによる配下組織データのアクセス許可
- ロール別アクセス制御の適用

### 4.2 性能考慮
- クラスタリングキー（device_id, collection_date）による検索最適化
- 大量データ取得時のページネーション
- グラフ描画データ件数の制限（最大1000件等）
