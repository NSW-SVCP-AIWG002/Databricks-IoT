# Customer Dashboard 設計準備資料

## 概要

本ドキュメントは、Customer Dashboard（顧客ダッシュボード）機能の設計に先立ち、要件の整理、不足情報の分析、新設章の詳細を記載します。

---

## 1. 把握した要件

### 1.1 既存要件（FR-006: 分析機能）

#### FR-006-1: ダッシュボード表示

| 項目 | 内容 |
|------|------|
| 画面ID | DSH-001 |
| 画面パス | `/`, `/dashboard` |
| 概要 | ログイン後のホーム画面として機能 |
| 表示方式 | Databricksダッシュボードをiframe埋め込み |
| 負荷 | Compute Clusterにかかる |
| 共有設定 | 作成者以外も参照可能（Databricks共有機能使用） |

**ダッシュボードの表示内容:**
- リアルタイムデータ: センサーデータ、デバイスステータス一覧
- 履歴データ: 時系列グラフ、期間指定による絞り込み、複数デバイスの比較表示
- 最新アラート情報: 直近n件のアラート通知

#### FR-006-2: データアクセス制御

| 項目 | 内容 |
|------|------|
| アクセス範囲制限 | ユーザーごとにアクセス可能なデータの範囲が異なる |
| 実装方式 | 動的ビューを経由した参照制限 |

#### FR-006-3: 対話型AI機能

| 項目 | 内容 |
|------|------|
| 概要 | ダッシュボード埋め込みの対話型AI機能 |
| 実装方式 | Databricks Genieを使用 |
| 機能 | 自然言語での質問入力、AIによる回答生成 |

#### アクセス権限

| 機能 | システム保守者 | 管理者 | 販社ユーザ | サービス利用者 |
|------|--------------|--------|-----------|--------------|
| ダッシュボード表示 | ○ | ○ | ○ | ○ |

### 1.2 追加要件（グラフ作成・配置機能）

#### 概要

顧客がグラフを作成し、ダッシュボード画面に自由に配置する機能を実装する。

#### グラフ種類

| No | グラフ種類 | 説明 | Apache ECharts対応 |
|----|-----------|------|-------------------|
| 1 | 棒グラフ | 縦棒グラフ、横棒グラフ | bar |
| 2 | 時系列グラフ（折れ線グラフ） | 時間軸に沿ったデータ推移 | line |
| 3 | 帯グラフ | 積み上げ棒グラフ/面グラフ | bar (stack) / line (area) |
| 4 | 円グラフ | 構成比の表示 | pie |
| 5 | 表 | データテーブル表示 | カスタム実装（HTML table） |

#### 技術要件

- **グラフライブラリ**: Apache ECharts
- **配置方式**: グリッドレイアウト（ドラッグ＆ドロップ）
- **保存先**: OLTP DB（顧客ごとのダッシュボード設定）
- **データソース**: Unity Catalog（ゴールド層）のセンサーデータ

### 1.3 非機能要件

#### パフォーマンス要件

| 項目 | 指標 |
|------|------|
| 初期ページ読み込み時間 | 3秒以内 |
| ダッシュボード表示時間 | 10秒以内（Databricks処理含む） |
| API応答時間 | 500ms以内 |

#### セキュリティ要件

| 項目 | 内容 |
|------|------|
| データスコープ制限 | 組織階層に基づいたフィルタリング |
| 動的ビュー | Unity Catalogの動的ビューで参照制限 |
| 認証 | Databricks User認証（Entra ID連携） |

#### 互換性要件

| 項目 | 対応 |
|------|------|
| ブラウザ | Google Chrome |
| 画面サイズ | 1920×1080 |

## 2. 解析した不足情報

### 2.1 グラフ設定のデータモデル

**課題**: 要件資料にグラフ設定の保存方法が明記されていない

**解決策**: 以下のデータモデルを新規設計

```
dashboard_widgets テーブル
├── widget_id (PK)
├── user_id (FK)
├── organization_id (FK)
├── widget_name
├── widget_type (bar/line/pie/area/table)
├── chart_config (JSON)
├── data_source_config (JSON)
├── position_x
├── position_y
├── width
├── height
├── display_order
├── delete_flag
├── create_date
├── creator
├── update_date
├── modifier
```

### 2.2 グラフデータの取得方法

**課題**: グラフに表示するデータの取得元・クエリ方法が未定義

**解決策**:
- データソース: Unity Catalog（ゴールド層）のセンサーデータビュー
- クエリ方法: SQL Warehouse経由でのクエリ実行
- フィルタリング: 動的ビューによる組織階層フィルタリング

### 2.3 グラフ配置のレイアウト方式

**課題**: グラフの自由配置の実装方式が未定義

**解決策**:
- グリッドベースのレイアウト（12カラムグリッド）
- ウィジェットのドラッグ＆ドロップ
- リサイズ機能（最小/最大サイズ制限あり）

### 2.4 Databricksダッシュボードとの共存

**課題**: 既存のDatabricksダッシュボード（iframe埋め込み）とカスタムグラフの共存方法

**解決策**:
- タブ切り替え方式
  - タブ1: Databricksダッシュボード（iframe埋め込み）
  - タブ2: カスタムダッシュボード（顧客作成グラフ）
- または統合方式
  - カスタムグラフウィジェットとDatabricks iframeウィジェットを同一画面に配置

## 3. 新設した章の詳細

### 3.1 UI仕様書の新設セクション

#### (A) グラフウィジェット管理UI

**理由**: 追加要件「グラフ作成機能」に対応するため、グラフウィジェットの作成・編集UIが必要

**内容:**
- グラフ作成モーダル
  - グラフ種類選択
  - データソース選択（デバイス、センサー種別、期間）
  - グラフタイトル設定
  - 表示オプション設定
- グラフ編集モーダル
- グラフ削除確認モーダル

#### (B) ダッシュボードレイアウト管理UI

**理由**: 追加要件「自由に配置する機能」に対応するため、レイアウト管理UIが必要

**内容:**
- ウィジェット配置エリア（グリッドレイアウト）
- ドラッグ＆ドロップによる配置変更
- リサイズハンドル
- 編集モード切替（表示モード/編集モード）

#### (C) Apache EChartsグラフ表示仕様

**理由**: 追加要件で指定されたApache EChartsの各グラフ種類に対応する表示仕様が必要

**内容:**
- 棒グラフの表示仕様
- 折れ線グラフ（時系列）の表示仕様
- 帯グラフ（積み上げ）の表示仕様
- 円グラフの表示仕様
- 表の表示仕様

### 3.2 ワークフロー仕様書の新設セクション

#### (A) グラフ設定CRUD処理フロー

**理由**: グラフウィジェットの作成・読取・更新・削除の処理フローが必要

**内容:**
- グラフ作成ワークフロー
- グラフ更新ワークフロー
- グラフ削除ワークフロー

**内容:**
- データソース選択フロー

- データ変換（ECharts形式へ）
- キャッシュ戦略

#### (C) レイアウト保存処理フロー

**理由**: ウィジェット配置情報の保存処理が必要

**内容:**
- 配置変更検知
- 自動保存（デバウンス処理）
- 保存確認

#### (D) Apache ECharts連携仕様

**理由**: Apache EChartsライブラリとの連携方法の詳細が必要

**内容:**
- EChartsインスタンス管理
- オプション設定
- レスポンシブ対応
- データ更新（リアルタイム更新オプション）

## 4. 設計方針

### 4.1 画面構成方針

```
┌─────────────────────────────────────────────────────────────┐
│ ヘッダー: カスタムダッシュボード                               │
├─────────────────────────────────────────────────────────────┤
│ タブ切替: [Databricksダッシュボード] [カスタムダッシュボード]    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────┐ ┌─────────────────────┐           │
│  │ ウィジェット1       │ │ ウィジェット2       │           │
│  │ （棒グラフ）        │ │ （折れ線グラフ）    │           │
│  │                     │ │                     │           │
│  └─────────────────────┘ └─────────────────────┘           │
│                                                             │
│  ┌───────────────────────────────────────────┐             │
│  │ ウィジェット3                              │             │
│  │ （表）                                    │             │
│  └───────────────────────────────────────────┘             │
│                                                             │
│  [+ ウィジェット追加]                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 データフロー方針

```
[ユーザー操作]
     ↓
[Flask App (SSR)]
     ↓
[OLTP DB] ←→ ウィジェット設定取得/保存
     ↓
[Unity Catalog] ←→ センサーデータ取得（SQL Warehouse経由）
     ↓
[Apache ECharts] → グラフ描画（クライアントサイド）
```

### 4.3 セキュリティ方針

1. **データスコープ制限**: ユーザーの所属組織に基づくフィルタリング
2. **ウィジェットアクセス制御**: ユーザー自身が作成したウィジェットのみ編集可能
3. **SQLインジェクション対策**: プリペアドステートメント使用
4. **XSS対策**: Jinja2自動エスケープ + EChartsデータサニタイズ
