## 基本情報

| 項目 | 内容 |
|------|------|
| 画面ID | `CDS-001` |
| 画面名 | 顧客作成ダッシュボード表示画面 |
| 画面パス | `/customer-dashboard` |
| アクセス権限 | システム保守者, 管理者, 販社ユーザ, サービス利用者（すべてのロール） |
| 親画面 | グローバルメニュー（ダッシュボード） |
| 子画面 | ダッシュボード管理モーダル（CDS-002）、ガジェット追加モーダル（CDS-003）、登録・更新モーダル（CDS-004）、削除確認モーダル（CDS-005） |

### (3) データソース選択フォーム

**要素構成図**

```
┌──────────────────────────────────────┐
│　(3) データソース選択フォーム　　　　　│
├──────────────────────────────────────┤
│　(3.1) 組織　　　　　(3.2) デバイス　　│
│　┌──────────────┐　┌──────────────┐　│
│　│            ▼ │　│            ▼ │　│
│　└──────────────┘　└──────────────┘　│
└──────────────────────────────────────┘
```

**3-1: 組織選択**
- 概要: 組織でフィルタリング
- ラベル: 「組織」
- 選択肢:
  - DB: organization_masterから動的取得
- 選択肢表示制限:
  - ユーザーの所属組織とその下位組織のみを取得して表示
- 選択肢のソート順
  - organization_master.organization_idの昇順  

**3-2: デバイス選択**
- 概要: デバイスでフィルタリング
- ラベル: 「デバイス」
- 活性条件:
  - 組織選択で組織が選択されている場合のみ活性
  - 組織選択で組織が選択されていない場合は非活性（初期状態は非活性）
- 選択肢:
  - DB: device_masterから動的取得（組織選択の内容でフィルタリング）
- 選択肢表示制限:
  - ユーザーの所属組織とその下位組織に紐づくデバイスのみを取得して表示
- 選択肢のソート順
  - device_master.device_idの昇順

**グローバルフィルタの動作仕様:**

データソース選択フォームはグローバルフィルタとして機能し、ガジェットのデータ取得時のWHERE条件を動的に変更します。ガジェットの `data_source_config` 自体は変更しません。

| 操作 | 動作 |
|------|------|
| 組織を選択 | デバイス選択肢が更新される。データソース種別が「組織」のガジェットは、選択された組織のデータを表示（WHERE句の `organization_id` が変更） |
| デバイスを選択 | データソース種別が「デバイス」のガジェットは、選択されたデバイスのデータを表示（WHERE句の `device_id` が変更） |
| 組織・デバイスが未選択 | ガジェットにデータが表示されない（ガジェットの枠とタイトルのみ表示） |

- 選択状態は `dashboard_user_setting.organization_id` と `dashboard_user_setting.device_id` に永続化される

## 備考

**空状態のUI:**
- ダッシュボードが0件の場合: 特別なガイダンスメッセージは表示せず、空の状態で表示する。ユーザーはダッシュボード管理ボタンから追加操作を行う
- ガジェットが0件の場合: 特別なガイダンスメッセージは表示せず、空の状態で表示する。ユーザーはグループ設定メニューからガジェット追加操作を行う

- 画面サイズは1920×1080を基準に設計
- 画面左部にグローバルメニューカラムを表示
- Apache EChartsのバージョン: 5.x系を使用
- グラフの描画はクライアントサイドで実行（JavaScriptによるレンダリング）
- データ取得はAJAXリクエストで非同期実行
- 編集モードのドラッグ＆ドロップはHTML5 Drag and Drop APIを使用

- ユーザー操作に対する処理フロー、バリデーション実行タイミング、エラーハンドリングの詳細は [ワークフロー仕様書](./workflow-specification.md) を参照してください
