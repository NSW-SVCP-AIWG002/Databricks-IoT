## 2. 解析した不足情報

### 2-1. 10桁の桁割り仕様

**状況**: 要件資料に「10桁の数値表示ガジェット」とあるが、整数部・小数部の具体的な桁数が未定義。

**設計にて定義**:
- 10桁 = 数値の有効桁数（符号・小数点を除いた数字部分）が最大10桁であることを意味する
- 表示最大文字列: `-1234567890`（符号1 + 整数部10桁 = 11文字）または `-12345678.90`（符号+整数8+小数点+小数2）
- 整数部: 最大8桁（符号含む最大9文字）
- 小数部: ユーザー設定 0〜4桁
- 表示枠: 最大13文字（符号1 + 整数部8 + 小数点1 + 小数部4）に対応したフォントサイズで設計

### 2-2. NULL値（データ未取得時）の表示方式

**状況**: デバイスからデータが届いていない・測定項目の値がNULLの場合の表示方法が未定義。

**設計にて定義**:
- 値がNULLの場合: `---` を表示
- データ取得失敗（DBエラー等）の場合: エラーアイコン + `取得失敗` テキストを表示

### 2-3. 小数点以下桁数のデフォルト・選択範囲

**状況**: 要件資料に小数点以下桁数の設定範囲が未定義。

**設計にて定義**:
- デフォルト: 2桁
- 選択範囲: 0桁 / 1桁 / 2桁 / 3桁 / 4桁

### 2-4. 数値の自動更新頻度

**状況**: 数値の自動更新（リアルタイム更新）間隔が本ガジェット固有に定義されていない。

**設計にて定義**:
- 顧客作成ダッシュボード共通の自動更新設定に従う
  - 自動更新ON時: ダッシュボード共通の更新間隔（30秒）で最新値を再取得
  - 自動更新OFF時: 手動更新（画面内に更新ボタンなし、ダッシュボード全体の手動更新で対応）

### 2-5. gadget_type_master への初期データ

**状況**: 数値表示ガジェット種別が `gadget_type_master` に未登録（新規追加が必要）。

**設計にて定義**:

| カラム | 値 |
|--------|-----|
| gadget_type_name | 数値表示 |
| gadget_type_slug | numerical-display |
| data_source_type | 1（デバイス単位） |
| gadget_image_path | static/images/gadget_numerical_display.png |
| gadget_description | センサーデータの最新値を1項目・数値ラベルで表示するガジェット |

## 3. 新設した章の詳細

### 3-1. UI仕様書 §「数値フォーマット仕様」章（新設）

**設置箇所**: `ui-specification.md` の「要素詳細」と「備考」の間に挿入

**新設理由**:
テンプレート（`ui-specification-template.md`）には数値の表示フォーマット定義の章が存在しない。
本ガジェットは「10桁の数値表示」という固有制約を持ち、桁割り・ゼロ埋め・負数表示・NULL表示・小数桁数といった表示仕様を明確化しなければ実装者が判断できない。
これらの仕様はUI要素の詳細には収まらず独立した章として設けることが適切と判断した。

**章の構成**:
- 表示フォーマット定義（整数部・小数部・符号の扱い）
- 桁数と表示例
- NULL値・エラー時の表示
- 小数点以下桁数の設定値と表示例
