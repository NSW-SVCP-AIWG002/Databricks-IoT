# メール通知履歴(mail-history) - 設計準備資料

## 1. 把握した要件

### 1.1 機能概要（FR-005-2 より）

**機能名:** メール通知履歴

**機能ID:** FR-005-2

**画面ID:**
- NTC-005: メール通知履歴一覧画面
- NTC-006: メール通知履歴参照画面

**概要:**
システム内で送信されたメール通知の履歴を参照する機能。アラート通知メール、招待メール、パスワードリセットメールなどの送信履歴を一覧表示し、詳細を確認できる。

**主要機能:**
- メール通知履歴の一覧表示
- メール通知履歴の検索・絞り込み
- メール通知履歴の詳細表示（モーダル）

**備考:**
- 参照専用機能（登録・更新・削除なし）
- アラート履歴（alert-history）とは別機能

### 1.3 検索条件項目

**注:** functional-requirements.md では「TODO」となっていたため、以下の検索条件を設計しました。

| No | 項目名 | 入力タイプ | 検索方式 | 備考 |
|----|--------|-----------|---------|------|
| 1 | メール種別 | MultiSelect | 完全一致 | アラート通知、招待メール、パスワードリセット等から選択 |
| 2 | 送信先メールアドレス | Text | 部分一致 | 送信先のメールアドレス |
| 3 | キーワード（件名・本文） | Text | 部分一致 | 件名または本文に含まれる文字列 |
| 4 | 送信日時（開始） | Date | 範囲検索 | 指定日時以降のデータを取得 |
| 5 | 送信日時（終了） | Date | 範囲検索 | 指定日時以前のデータを取得 |

**理由:** メール通知履歴を効率的に検索するため、メール種別、送信先、キーワード、日時範囲の検索条件を追加しました。

## 4. 設計方針

### 4.1 参照専用機能としての設計

メール通知履歴は参照専用機能（登録・更新・削除なし）のため、以下の設計方針を採用します:
- 一覧画面に登録・削除ボタンを配置しない
- モーダルは参照のみ（編集ボタンなし）
- データテーブルにチェックボックスを配置しない

### 4.2 データスコープ制限の実装

販社ユーザとサービス利用者は自社に紐づくデータのみ閲覧可能とするため、以下を実装します:
- OLTP DBのWHERE句にデータスコープ制限を追加
- `organization_id = current_user.organization_id`

### 4.3 検索機能の充実

メール通知履歴を効率的に検索できるよう、以下の検索条件を実装します:
- メール種別（MultiSelect）
- 送信先メールアドレス（部分一致）
- キーワード（件名・本文、部分一致）
- 送信日時（範囲検索）

### 4.4 パフォーマンス最適化

大量のメール送信履歴を扱うため、以下を実装します:
- ページネーション（1ページあたり25件）
- インデックスの活用（organization_id, sent_at, mail_type）
- 検索条件による絞り込み
