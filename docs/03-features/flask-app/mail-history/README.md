## 概要

メール通知履歴機能は、Databricks IoTシステム内で送信されたメール通知の履歴を参照する機能です。アラート通知メール、招待メール、パスワードリセットメールなどの送信履歴を一覧表示し、詳細を確認できます。

**注:** 本機能は参照専用機能であり、登録・更新・削除操作は提供しません。

**機能ID:** FR-005-2

## データモデル

### メール送信履歴 (mail_history)

#### テーブル設計

| カラム名 | 論理名 | データ型 | 必須 | 備考 |
|----------|-------|---------|------|------|
| mail_history_id | メール送信履歴ID | INT | ○ | 主キー |
| mail_history_uuid | メール送信履歴UUID | VARCHAR(36) | ○ | UUID（外部公開用一意識別子） |
| mail_type | メール種別ID | INT | ○ | 外部キー: mail_type_master.mail_type_id |
| sender_email | 送信元メールアドレス | VARCHAR(254) | ○ | 送信元のメールアドレス |
| recipients | 送信先メールアドレス | JSON | ○ | 送信先のメールアドレス（JSON形式） |
| subject | メール件名 | VARCHAR(500) | ○ | メールの件名 |
| body | メール本文 | TEXT | ○ | メールの本文 |
| sent_at | 送信日時 | DATETIME | ○ | メール送信日時 |
| user_id | 関連ユーザーID | INT | - | 外部キー: user_master.user_id |
| organization_id | 関連組織ID | INT | - | 外部キー: organization_master.organization_id（データスコープ制限用） |
| create_date | 作成日時 | DATETIME | ○ | レコード作成日時 |
| creator | 作成者 | INT | ○ | レコード作成者のユーザID |
| update_date | 更新日時 | DATETIME | - | レコード最終更新日時 |
| modifier | 更新者 | INT | - | レコード更新者のユーザID |

#### インデックス

| インデックス名 | カラム | 種別 | 目的 |
|---------------|--------|-----|------|
| PRIMARY | mail_history_id | 主キー | メール送信履歴の一意識別 |
| uk_mail_history_uuid | mail_history_uuid | UNIQUE INDEX | UUID検索高速化、一意性保証 |
| idx_organization_id | organization_id | INDEX | データスコープ制限用 |
| idx_sent_at | sent_at | INDEX | 日時範囲検索高速化 |
| idx_mail_type | mail_type | INDEX | メール種別検索高速化 |

## アクセス権限

| 機能 | システム保守者 | 管理者 | 販社ユーザ | サービス利用者 |
|------|---------------|--------|-----------|---------------|
| メール通知履歴一覧・検索 | ○ | ○ | ○ | ○ |
| メール通知履歴参照 | ○ | ○ | ○ | ○ |

**凡例:**
- **○**: 利用可能（すべてのデータにアクセス可能）
- **-**: 利用不可

**データスコープ制限:**
- 全ユーザ：自身の組織の`organization_id`を持つメール送信履歴のみ表示可能
