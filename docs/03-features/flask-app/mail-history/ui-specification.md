## 要素詳細

### (2) 検索フォーム

**要素構成図:**

```
┌─────────────────────────────────────────────────────────────────────┐
│ (2) 検索フォーム                                                  ▼  │
├─────────────────────────────────────────────────────────────────────┤
│ (2.1) メール種別  (2.2) 送信日時(開始) (2.3) 送信日時(終了) (2.4) キーワード│
│ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌────────────┐│
│ │▼ 選択        │ │YYYY/MM/DD    │ │YYYY/MM/DD    │ │(件名・本文)││
│ └──────────────┘ └──────────────┘ └──────────────┘ └────────────┘│
│                                                                     │
│ (2.5) [検索]                                                        │
└─────────────────────────────────────────────────────────────────────┘
```

**2-1: メール種別ドロップダウン**
- 概要: メール送信履歴をメール種別でフィルタリングするための選択項目。複数選択可能
- ラベル: 「メール種別」
- BEMクラス: `.form-select form-select--multiple`
- 選択肢:
  - アラート通知
  - 招待メール
  - パスワードリセット
  - システム通知

**HTML例:**
```html
<div class="form-group">
  <label for="mailTypes" class="form-label">メール種別</label>
  <select id="mailTypes" name="mail_type_ids" class="form-select form-select--multiple" multiple size="4">
    <option value="1">アラート通知</option>
    <option value="2">招待メール</option>
    <option value="3">パスワードリセット</option>
    <option value="4">システム通知</option>
  </select>
</div>
```

**2-2: 送信日時（開始）**
- 概要: 指定日時以降のメール送信履歴を取得するための日付選択項目
- ラベル: 「送信日時（開始）」
- BEMクラス: `.form-input`
- 入力タイプ: date

**HTML例:**
```html
<div class="form-group">
  <label for="sentAtStart" class="form-label">送信日時（開始）</label>
  <input type="date" id="sentAtStart" name="sent_at_start" class="form-input" value="{{ sent_at_start }}">
</div>
```

**2-3: 送信日時（終了）**
- 概要: 指定日時以前のメール送信履歴を取得するための日付選択項目
- ラベル: 「送信日時（終了）」
- BEMクラス: `.form-input`
- 入力タイプ: date

**HTML例:**
```html
<div class="form-group">
  <label for="sentAtEnd" class="form-label">送信日時（終了）</label>
  <input type="date" id="sentAtEnd" name="sent_at_end" class="form-input" value="{{ sent_at_end }}">
</div>
```

**2-4: キーワード入力**
- 概要: メール件名または本文に含まれる文字列でフィルタリングするための入力項目。部分一致検索
- ラベル: 「キーワード（件名・本文）」
- BEMクラス: `.form-input`

**HTML例:**
```html
<div class="form-group">
  <label for="keyword" class="form-label">キーワード（件名・本文）</label>
  <input type="text" id="keyword" name="keyword" class="form-input" maxlength="255">
</div>
```

### (4) データテーブル

**要素構成図:**

```
┌─────────────────────────────────────────────────────────────────────────┐
│ (4) メール通知履歴一覧テーブル                                            │
├────┬───────────┬──────────────┬──────────┬────────────┤
│(4.1)│  (4.2)    │   (4.3)      │  (4.4)   │   (4.5)    │
│メール│件名       │本文（抜粋）  │送信日時  │ アクション │
│種別 │           │              │          │            │
├────┼───────────┼──────────────┼──────────┼────────────┤
│[アラ│【緊急】   │デバイスID... │2025/12/10│[詳細]      │
│ート]│アラート   │              │  10:30   │            │
├────┼───────────┼──────────────┼──────────┼────────────┤
│[招待│【招待】   │ユーザー登録... │2025/12/09│[詳細]      │
│メール│システム   │              │  15:20   │            │
└────┴───────────┴──────────────┴──────────┴────────────┘
                 (4.6) [< 1 2 3 ... 10 >] 全250件
```

**カラム幅:**

| No | カラム名 | 幅 |
|----|---------|-----|
| 4-1 | メール種別 | 15% |
| 4-2 | 件名 | 30% |
| 4-3 | 本文（抜粋） | 35% |
| 4-4 | 送信日時 | 12% |
| 4-5 | アクション | 8% |

**テーブル設定:**
- テーブル全体の最小幅: 1000px（この幅を下回る場合は横スクロール）
- 1ページあたりの表示件数: 25件（固定）
- デフォルトソート: 送信日時（降順）、メール履歴ID（昇順）
- ソート仕様: すべてのソート項目において、第2ソートキーとしてメール履歴ID（昇順）を使用し、ページング時の並び順を一定に保つ
- ローディング中の表示: スケルトンスクリーン

**4-1: メール種別カラム**
- 概要: メールの種別をバッジ形式で表示
- BEMクラス: `.table__badge`
- スタイル:
  - アラート通知: 赤色バッジ（`.table__badge--danger`）
  - 招待メール: 青色バッジ（`.table__badge--info`）
  - パスワードリセット: 黄色バッジ（`.table__badge--warning`）
  - システム通知: 緑色バッジ（`.table__badge--success`）

**HTML例:**
```html
<span class="table__badge table__badge--danger">アラート通知</span>
<span class="table__badge table__badge--info">招待メール</span>
```

**CSS定義（画面固有）:**
```css
/* メール種別バッジ（UI共通仕様のバッジを拡張） */
.table__badge--danger {
  background-color: var(--color-danger);
  color: var(--color-white);
}

.table__badge--warning {
  background-color: var(--color-warning);
  color: var(--color-gray-900);
}

.table__badge--success {
  background-color: var(--color-success);
  color: var(--color-white);
}
```

**4-2: 件名カラム**
- 概要: メールの件名を表示。一覧では省略表示（最大50文字）
- BEMクラス: `.table__truncate`
- 51文字以上の場合: 「...」で省略

**4-3: 本文（抜粋）カラム**
- 概要: メールの本文を表示。一覧では省略表示（最大100文字）
- BEMクラス: `.table__truncate`
- 101文字以上の場合: 「...」で省略
- 改行は削除してスペースに置換

**4-4: 送信日時カラム**
- 概要: メール送信日時を表示
- 表示形式: `YYYY/MM/DD HH:mm`

**4-5: アクションボタンカラム**
- 概要: メール通知履歴の詳細を表示するボタン
- BEMクラス: `.table__action-btn`
- ボタン: 「詳細」
- クリック時: メール通知履歴参照モーダル（5）を開く
- メール送信履歴UUIDをリクエストし、詳細情報を取得する


**HTML例:**
```html
<button class="table__action-btn" onclick="openMailHistoryModal('mail_history_uuid')">詳細</button>
```

**ソートインジケーター:**
- デフォルト: ↕（ソート可能を示す）
- 昇順: ↑
- 降順: ↓

**5-1: メール種別**

- スタイル: データテーブルと同じバッジカラー

**5-2: 送信元メールアドレス**
- 概要: メールの送信元メールアドレスを表示
- ラベル: 「送信元メールアドレス」

**5-3: 送信先メールアドレス**
- 概要: メールの送信先メールアドレスを表示
- ラベル: 「送信先メールアドレス」
- データソース: DB `mail_history.recipients` (JSON型)
- 表示形式: `recipients.to` 配列の要素をカンマ区切りで表示（例: `user1@example.com, user2@example.com`）

**5-4: 送信日時**
- 概要: メール送信日時を表示
- ラベル: 「送信日時」
- 表示形式: `YYYY/MM/DD HH:mm`

**5-5: メール件名**
- 概要: メールの件名を全文表示
- ラベル: 「メール件名」

**5-6: メール本文**
- 概要: メールの本文を全文表示。改行を保持
- ラベル: 「メール本文」
- スタイル:
  - 読み取り専用テキストエリア
  - 最大高さ: 400px（スクロール可能）
  - 改行を保持（`white-space: pre-wrap`）

  border-radius: var(--border-radius-base);
