# アラート設定管理機能 設計準備資料

## 1. 把握した要件

### 1.1 機能概要

**機能ID**: FR-004-3
**機能名**: アラート設定管理
**カテゴリ**: マスタ管理機能（Flaskアプリケーション）

アラート設定管理機能は、IoTデバイスから収集されるデータに対する異常検出条件を設定・管理する機能です。

### 1.2 サブ機能一覧

| 機能ID | 機能名 | 画面ID | 概要 |
|--------|--------|--------|------|
| FR-004-3-1 | アラート設定一覧・検索 | ALT-001 | アラート設定の一覧表示・検索・削除 |
| FR-004-3-2 | アラート設定参照 | ALT-004 | アラート設定の詳細情報表示 |
| FR-004-3-3 | アラート設定登録 | ALT-002 | アラート設定の登録 |
| FR-004-3-4 | アラート設定編集 | ALT-003 | アラート設定の変更 |
| FR-004-3-5 | アラート設定削除 | ALT-001 | アラート設定の論理削除（一覧画面から実行） |

### 1.3 画面仕様

| 画面ID | 画面名 | スラッグ | 概要 | HTTPメソッド |
|--------|--------|---------|------|-------------|
| ALT-001 | アラート設定一覧画面 | /alert/alert-setting | 一覧・検索/削除 | GET (一覧), POST (削除) |
| ALT-002 | アラート設定登録画面 | /alert/alert-setting/register | 登録フォーム表示・登録実行 | GET (フォーム), POST (実行) |
| ALT-003 | アラート設定更新画面 | /alert/alert-setting/update | 更新フォーム表示・更新実行 | GET (フォーム), POST (実行) |
| ALT-004 | アラート設定参照画面 | /alert/alert-setting/detail/:id | 詳細情報表示 | GET |

### 2.3 アラート発生条件・復旧条件の入力仕様

**要件書の状態**:
- 表示項目として「アラート発生条件」「アラート復旧条件」が定義されている
- 具体的な入力方式が未定義

**推奨する入力仕様**:

**方式A: ルールビルダー形式**:
- 条件式を GUI で構築
- 例: 「温度 > 30℃ かつ 湿度 > 80%」
- メリット: ユーザーフレンドリー、入力ミス防止
- デメリット: 実装コストが高い

**方式B: テキスト入力形式**:
- 条件式をテキストで入力
- 例: `temperature > 30 AND humidity > 80`
- メリット: 実装が容易
- デメリット: 構文エラーが発生しやすい、バリデーションが複雑

**方式C: プリセット条件選択形式**:
- よく使われる条件をプリセットとして用意
- カスタム条件はテキスト入力
- メリット: 使いやすさと柔軟性のバランスが良い
- デメリット: プリセット条件の定義が必要

**決定が必要な事項**:
- どの方式を採用するか、要件定義者と協議が必要
- 暫定版では「方式B: テキスト入力形式」を採用し、後続フェーズで改善を検討

## 4. 設計方針

### 4.1 基本方針

1. **テンプレート遵守**:
   - ui-specification-template.md に従った UI仕様書作成
   - workflow-specification-template.md に従ったワークフロー仕様書作成

2. **要件完全反映**:
   - FR-004-3-1〜5 のすべての機能を実装
   - アクセス権限マトリクスを正確に反映
   - 非機能要件を満たす設計

3. **データスコープ制限**:
   - WHERE句による organization_id フィルタリング
   - 販社ユーザ、サービス利用者は自社データのみアクセス可能

4. **Flask SSR設計**:
   - REST APIではなく Flask SSR（Server-Side Rendering）
   - HTMLレスポンス（`render_template()`）またはリダイレクト（`redirect()`）
   - モーダルダイアログで登録・更新・参照画面を表示

5. **CRUD画面テンプレート適用**:
   - 一覧画面: 検索フォーム、データテーブル、ページング
   - 登録・更新画面: モーダルダイアログ、バリデーション、確認モーダル
   - 削除機能: チェックボックス選択、確認モーダル、論理削除
