# ダッシュボード機能 設計準備資料

## 概要

このファイルは、ダッシュボード機能（FR-006）の設計資料作成にあたり、把握した要件、解析した不足情報、および新設した章の詳細を記載します。

---

## 1. 把握した要件

### 1.1 機能要件（functional-requirements.md）

#### FR-006: 分析機能

**FR-006-1: ダッシュボード表示**
- ログイン後のホーム画面として機能
- 表示方式: Databricksダッシュボードをiframe埋め込み
- 負荷: Compute Clusterにかかる
- 共有設定: 作成者以外も参照可能（Databricks共有機能使用）

**ダッシュボードの表示内容:**
- リアルタイムデータ: センサーデータ、デバイスステータス一覧
- 履歴データ: 時系列グラフ、期間指定による絞り込み、複数デバイスの比較表示
- 最新アラート情報: 直近n件のアラート通知

**表示内容の制御:**
- ユーザーのロールに応じてDatabricks側の動的ビューによりアクセス可能なデータが制限

**FR-006-2: データアクセス制御**
- アクセス範囲制限: ユーザーごとにアクセス可能なデータの範囲が異なる
- 実装方式: 動的ビューを経由した参照制限

**FR-006-3: 対話型AI機能**
- ダッシュボード埋め込みの対話型AI機能（Databricks Genie使用）
- 個別の画面としては作成せず、ダッシュボードの一部として提供
- 機能:
  - 自然言語での質問入力
  - AIによる回答生成
  - データに基づいた回答の提供
- モデル認識の改善:
  - 課題: 日本語の質問で英語の物理名で作ったテーブルをモデルが認識できない場合あり
  - 対策: 日本語名の動的ビューを作成してモデルが認識しやすくする

**FR-006-4: アクセス権限**

| 機能 | システム保守者 | 管理者 | 販社ユーザ | サービス利用者 |
|------|---------------|--------|-----------|---------------|
| ダッシュボード表示 | ○ | ○ | ○ | ○ |

#### 画面情報（画面一覧）

| 画面ID | カテゴリ | 画面名 | スラッグ | 概要 | 備考 |
|--------|---------|--------|----------|------|------|
| DSH-001 | ダッシュボード | ダッシュボード表示 | /, /dashboard | ログイン後の初期画面。Databricks側で作成した単一のダッシュボードをiframe埋め込み表示 | 対話型AI機能を含む、ホーム画面兼用 |

#### 画面共通要件

- 画面サイズは1920×1080
- 画面左部にグローバルメニューカラムを表示
  - カラム上部にはロゴ、下部にはアカウント情報を表示
  - メニューの各要素として初期状態では機能分類を表示
  - マウスホバー時にサブメニューを展開し、各機能分類の子要素を表示
- 画面の機能名は画面左上最上部に表示

### 1.2 非機能要件（non-functional-requirements.md）

**NFR-PERF-003: Web画面応答性能**
- ダッシュボード表示時間: **10秒以内**（Databricks側の処理時間含む）

**NFR-SEC-001: 認証・認可**
- ダッシュボード表示: すべてのロールがアクセス可能
- Databricks User認証（Databricks標準機能、Flaskアプリでは実装しない）

**NFR-SEC-007: データスコープ制限要件**
- 組織階層に基づいたデータアクセス範囲を制限
- 動的ビュー (Dynamic Views) を使用して組織階層ベースでデータをフィルタリング

### 1.3 技術要件（technical-requirements.md）

**TR-DB-001: Databricks Platform**
- Databricksダッシュボードをiframe埋め込み
- Unity Catalog経由でゴールド層のデータを取得

**TR-FE-001: Flask Application**
- Flask SSR（Server-Side Rendering）
- Jinja2テンプレートエンジン

**TR-SEC-001: 認証・認可**
- ログイン認証はDatabricks標準機能
- Flaskアプリはリバースプロキシヘッダからユーザー情報を取得

## 2. 解析した不足情報

### 2.1 Databricksダッシュボード仕様（未確定項目）

| No | 項目 | 状態 | 備考 |
|----|------|------|------|
| 1 | ダッシュボードのURL | 未確定 | Databricks側で作成後に確定 |
| 2 | iframe埋め込み時の認証方式 | 未確定 | Databricks側の共有設定に依存 |
| 3 | 最新アラート表示件数（n件） | 未確定 | 要件では「直近n件」と記載 |
| 4 | 期間指定の範囲（履歴データ） | 未確定 | デフォルト期間、最大期間が未定義 |
| 5 | 対話型AIの日本語対応状況 | 課題あり | 日本語名動的ビュー作成が対策として記載 |

### 2.2 設計上の前提

上記の未確定項目については、以下の前提で設計を進めます：

1. **ダッシュボードURL**: 環境変数で設定可能な構成とする
2. **iframe認証**: Databricks側の共有設定（パブリック共有/組織内共有）を利用
3. **最新アラート表示件数**: 10件をデフォルトとして設計（変更可能）
4. **期間指定**: Databricksダッシュボード側のウィジェット機能で実現
5. **対話型AI**: Databricks Genie埋め込みとして設計

## 3. 新設した章の詳細

### 3.1 UI仕様書での新設章

#### 「iframe埋め込み仕様」セクション

**新設理由:**
- 標準のCRUD画面テンプレートには存在しないiframe埋め込み特有の仕様を記載する必要がある
- Databricksダッシュボードの埋め込みに関する技術的詳細を明確化するため

**記載内容:**
- iframeのサイズ・配置仕様
- ローディング状態の表示
- エラー時のフォールバック表示
- レスポンシブ対応

### 3.2 ワークフロー仕様書での新設章

#### 「Databricksダッシュボード連携」セクション

**新設理由:**
- Flask SSRとDatabricksダッシュボード間の連携フローを明確化するため
- 二重認証フロー（Flask App → Databricks Dashboard）の説明が必要

**記載内容:**
- Flask側の処理フロー
- iframe内でのDatabricks認証フロー
- エラーハンドリング

## 4. 設計方針

### 4.1 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ (1) グローバルメニュー │ (2) メインコンテンツエリア          │
│                        │                                     │
│  ロゴ                  │  (2.1) ヘッダー                     │
│                        │  ┌─────────────────────────────────┐│
│  ダッシュボード        │  │ ダッシュボード                  ││
│  管理機能 ▶            │  └─────────────────────────────────┘│
│  アラート機能 ▶        │                                     │
│  通知機能 ▶            │  (2.2) Databricksダッシュボード     │
│  CSVインポート         │  ┌─────────────────────────────────┐│
│  アカウント ▶          │  │                                 ││
│                        │  │  <iframe>                       ││
│                        │  │    Databricks Dashboard         ││
│                        │  │    (リアルタイムデータ)         ││
│                        │  │    (履歴データ)                 ││
│  ─────────────         │  │    (最新アラート)               ││
│  アカウント情報        │  │    (対話型AI)                   ││
│                        │  │                                 ││
│                        │  └─────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

### 4.2 技術アーキテクチャ

1. **Flask側の責務**
   - ページレイアウト（グローバルメニュー、ヘッダー）のレンダリング
   - iframe埋め込み用のHTMLテンプレート提供
   - ダッシュボードURLの環境変数管理

2. **Databricks側の責務**
   - ダッシュボードコンテンツの表示
   - データの取得・可視化（ゴールド層）
   - 動的ビューによるデータスコープ制限
   - 対話型AI機能（Databricks Genie）

### 4.3 認証フロー

```
ユーザー → Flask App認証 → ダッシュボード画面表示
         ↓
         iframe内でDatabricksダッシュボードアクセス
         ↓
         Databricks認証（Entra ID）
         ↓
         ダッシュボードコンテンツ表示
```
