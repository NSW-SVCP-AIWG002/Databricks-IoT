### (2) 検索フォーム

**要素構成図:**

```
┌─────────────────────────────────────────────────────────┐
│ (2) 検索フォーム                                         │
├─────────────────────────────────────────────────────────┤
│ (2.1) キーワード          (2.2) 権限                    │
│ ┌─────────────────────┐  ┌─────────────────────┐      │
│ │                     │  │▼ 選択してください   │      │
│ └─────────────────────┘  └─────────────────────┘      │
│                                                         │
│ (2.3) 所属グループ                                      │
│ ┌─────────────────────┐                                │
│ │▼ 選択してください   │                                │
│ └─────────────────────┘                                │
│                                                         │
│ (2.4) [検索]  (2.5) [クリア]                           │
└─────────────────────────────────────────────────────────┘
```

**2-1: キーワード入力**
- 概要: ユーザー名およびメールアドレスでキーワードに入力した値で合致するものを表示するのに利用
- ラベル: 「キーワード」
- プレースホルダー: 「ユーザ名またはメールアドレスで検索」
- 入力中のリアルタイムバリデーション: なし

**2-2: 権限ドロップダウン**
- 概要: ユーザーの権限でフィルタリングするための選択項目。複数選択可能
- ラベル: 「権限」
- 選択肢:
  - NSW管理者（表示しない - 隠しユーザー）
  - ユーザ管理者
  - 一般ユーザ

**2-3: 所属グループドロップダウン**
- 概要: ユーザーが所属するグループでフィルタリングするための選択項目。複数選択可能
- ラベル: 「所属グループ」

**バリデーション:**

| 項目 | バリデーションルール | エラーメッセージ |
|------|---------------------|-----------------|
| 2-1 キーワード | 最大255文字 | 「キーワードは255文字以内で入力してください」 |

**注:** サーバー側バリデーションの詳細は、各API仕様書を参照してください。

### (4) データテーブル

**要素構成図:**

```
┌─────────────────────────────────────────────────────────────────────────┐
│ (4) データテーブル                                                       │
├─────┬──────────┬─────────────┬────────┬──────────┬──────────────┐
│(4.1)│  (4.2)   │   (4.3)     │ (4.4)  │  (4.5)   │    (4.6)     │
│ユーザ│メール    │   権限      │グループ│最終ログイン│ アクション   │
│名   │アドレス   │             │        │          │              │
├─────┼──────────┼─────────────┼────────┼──────────┼──────────────┤
│山田 │yamada@...│[管理者]     │グループA│2025/11/01│[詳細][編集]  │
│太郎 │          │             │        │  10:30   │              │
├─────┼──────────┼─────────────┼────────┼──────────┼──────────────┤
│佐藤 │sato@...  │[一般]       │グループB│2025/10/30│[詳細][編集]  │
│花子 │          │             │        │  15:20   │              │
└─────┴──────────┴─────────────┴────────┴──────────┴──────────────┘
                 (4.7) [< 1 2 3 ... 10 >] 全250件
```

**カラム幅:**

| No | カラム名 | 幅 |
|----|---------|-----|
| 4-1 | ユーザ名 | 20% |
| 4-2 | メールアドレス | 25% |
| 4-3 | 権限 | 15% |
| 4-4 | 所属グループ | 20% |
| 4-5 | 最終ログイン日時 | 15% |
| 4-6 | アクション | 10% |

**注:** 幅は相対値（%）または grid（fr）で指定してください。px指定は避けてください。

**テーブル設定:**
- テーブル全体の最小幅: 1200px（この幅を下回る場合は横スクロール）
- 1ページあたりの表示件数: 25件（固定）
- 最大表示件数: 1000件
- デフォルトソート: ユーザ名（昇順）
- 空の場合の表示: 「ユーザが見つかりませんでした」
- ローディング中の表示: スケルトンスクリーン

**4-1: ユーザ名カラム**
- 概要: ユーザーの名前を表示し、クリックすることでユーザー詳細画面へ遷移するリンク
- スタイル: テキストリンク（`appTheme.colors.brand_dark_pink`）
- クリック時: ユーザ詳細画面へ遷移（URL: `/users/:userId`）

**4-4: 所属グループカラム**
- 概要: ユーザーが所属するグループ名を表示。複数ある場合はカンマ区切りで最大3件まで表示
- 4件以上の場合: 「グループA, グループB, グループC, +2」のように表示

**ソートインジケーター:**
- デフォルト: ↕（ソート可能を示す）
- 昇順: ↑
- 降順: ↓

### (6) ユーザ登録モーダル

**モーダル仕様:**

| 項目 | 内容 |
|------|------|
| タイトル | 「ユーザ登録」 |
| サイズ | 中（600px幅） |
| 閉じる方法 | ×ボタン、背景クリック、ESCキー |

**要素構成図:**

```
┌─────────────────────────────────────────┐
│ ユーザ登録                           [×]│
├─────────────────────────────────────────┤
│                                         │
│ (6.1) メールアドレス *                  │
│ ┌─────────────────────────────────────┐ │
│ │ example@example.com                 │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ (6.2) ユーザ名 *                        │
│ ┌─────────────────────────────────────┐ │
│ │ 山田 太郎                           │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ (6.3) 権限 *                            │
│ ○ ユーザ管理者  ○ 一般ユーザ           │
│                                         │
│ (6.4) 所属グループ                      │
│ ┌─────────────────────────────────────┐ │
│ │ ▼ 選択してください                  │ │
│ └─────────────────────────────────────┘ │
│                                         │
├─────────────────────────────────────────┤
│         (6.6) [キャンセル]  (6.5) [登録]│
└─────────────────────────────────────────┘
```

**6-1: メールアドレス**
- 概要: ユーザーのログインに使用するメールアドレスを入力。システム内で一意である必要がある
- ラベル: 「メールアドレス」
- プレースホルダー: 「example@example.com」
- 入力タイプ: email

**6-2: ユーザ名**
- 概要: システム上で表示されるユーザーの名前を入力
- ラベル: 「ユーザ名」
- プレースホルダー: 「山田 太郎」

**6-3: 権限**
- 概要: ユーザーに付与する権限を選択。ユーザー管理者または一般ユーザーのいずれか
- ラベル: 「権限」
- 選択肢:
  - ユーザ管理者
  - 一般ユーザ

**6-4: 所属グループ**
- 概要: ユーザーが所属するグループを選択。複数選択可能で、未選択も可
- ラベル: 「所属グループ」

**バリデーション:**

| 項目 | バリデーションルール | エラーメッセージ |
|------|---------------------|-----------------|
| 6-1 メールアドレス | 必須 | 「メールアドレスを入力してください」 |
| 6-1 メールアドレス | メールアドレス形式（RFC 5322） | 「有効なメールアドレスを入力してください」 |
| 6-1 メールアドレス | 最大255文字 | 「メールアドレスは255文字以内で入力してください」 |
| 6-2 ユーザ名 | 必須 | 「ユーザ名を入力してください」 |
| 6-2 ユーザ名 | 最小1文字 | 「ユーザ名は1文字以上入力してください」 |
| 6-2 ユーザ名 | 最大255文字 | 「ユーザ名は255文字以内で入力してください」 |
| 6-3 権限 | 必須 | 「権限を選択してください」 |

**注:** サーバー側バリデーションの詳細は、各API仕様書を参照してください。

**ボタン配置:**
- 6-5 登録ボタン: 右側、プライマリボタン、ユーザ登録API呼び出し
- 6-6 キャンセルボタン: 左側、セカンダリボタン、モーダルを閉じる
